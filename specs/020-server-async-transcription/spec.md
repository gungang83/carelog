# Feature Specification: 서버 비동기 전사

**Feature Branch**: `020-server-async-transcription` (작업: `claude/eo-sso-token-verify-w9v931`)
**Created**: 2026-06-29 · **Status**: Implemented
**Extends**: spec 008(상담보드)·009(음성 보관)·012(알림)·016(자동저장). 클라 탭 의존 제거.

## Summary

"상담 종료 및 저장"을 **서버 위임**으로 전환. 종료 시 **음성만 Storage에 올리고 즉시 종료**하고,
전사·요약은 **서버 백그라운드 워커**가 처리한다. 사용자는 **탭을 닫거나 폰을 잠가도** 됨 —
완료되면 알림(spec 012)이 온다. 클라이언트 탭 생명주기 의존(=18분 크래시 근본원인) 제거.

- **즉시 플레이스홀더**: 종료 즉시 상담 레코드를 "🎙️ 서버에서 전사 중…"으로 생성 → 리스트에 바로 뜸(저장 확신 + 진행 가시성). 워커가 완료 시 본문을 채운다.
- **워커**: 매 분 cron이 pending job을 집어 음성 다운로드 → 전사(Whisper)+요약(Claude) → 상담 본문 갱신 → 완료 알림. 실패는 3회 재시도 후 실패 표시(음성 보관, 재시도 가능).
- **"상담 종료"(검토 경로)는 불변** — 검토 후 수동 저장 원하면 그대로.

## Requirements

- **FR-001** "상담 종료 및 저장": 음성(청크면 이어붙임)을 서버에 올리고 플레이스홀더 상담 생성 + job(pending) 등록 → 보드 즉시 종료. 대기 0·탭 종료 무관.
- **FR-002** 워커 cron(`/api/cron/process-transcriptions`, 매 분): pending job 원자적 클레임 → 음성 다운로드 → `runServerTranscription`(세션 없이) → 본문 갱신 → job done → 크레딧·토큰 기록 → 완료 알림.
- **FR-003** 사용자가 미리 입력한 본문(prefix)은 전사 앞에 보존.
- **FR-004** 실패: 3회까지 재시도(다시 pending), 초과 시 error + 본문에 "전사 실패(음성 보관, 재시도)" + 실패 알림. **유실 0.**
- **FR-005** 체어 미선택·빈 녹음은 등록 안 함(안내). 등록 실패 시 IndexedDB 임시본 보존 + 안내 + 로그.
- **FR-006** 인증: 워커는 CRON_SECRET Bearer(자동) 또는 슈퍼어드민 세션(수동 `?`). service_role만 job 접근(RLS 정책0).
- **FR-007** 서버 워커는 통짜 전사(Whisper 1회, ≤25MB). 엔진별 요약 프롬프트(detailed/dental 등)는 존중.

## Success Criteria

- **SC-001** 종료 즉시 보드가 닫히고 다음 환자로 이동 가능(탭 닫기·폰 잠금에도 완료).
- **SC-002** 완료 시 알림 + 상담 본문 자동 반영.
- **SC-003** 어떤 실패도 음성·작성물 유실 없음(재시도 가능).

## Out of Scope

- 완료 시 리스트 **실시간 자동 갱신**(현재는 알림 클릭/재진입 시 반영) — 후속(consultation realtime).
- 서버 워커의 청크 실패격리(통짜 전사로 단순화; 25MB 초과 초장시간은 후속).
- 실시간 스트리밍 전사.

## Notes

- Vercel cron `* * * * *`(매 분) — 플랜상 분단위 미지원이면 `*/2`로 조정.
- OpenAI/Anthropic 키가 서버(Vercel)에 있어야 워커 동작. 실기기·실키 검증 필요.
