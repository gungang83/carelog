# Feature Specification: 상담 기록 열람·검색·필터 + 환자별 간략 보기 (Records Browser)

**Feature Branch**: `011-records-browser`
**Created**: 2026-06-28
**Status**: Draft
**Input**: 예미안 파일럿 260628 피드백(A5+A6). 상담 카드가 줄줄이 쌓여 보기·찾기 힘듦 → 전체 열람 + 정렬/필터/검색 + 환자별 간략 보기.

> 용어 규칙(의료법): 산출물은 "상담 기록". "진료기록/의무기록" 미사용.

## User Scenarios & Testing

### US1 — 상담 기록 전체 검색·필터 (Priority: P1)
직원이 쌓인 상담 기록을 키워드·날짜·체어·연결상태로 좁혀 빠르게 찾는다.
**Acceptance**:
1. Given 상담이 많이 쌓였을 때, When '전체보기'로 들어가 키워드(예: "임플란트")를 입력하면, Then 본문에 그 키워드가 든 상담만 보인다.
2. Given 미연결만 보고 싶을 때, When 상태 필터 '미연결'을 고르면, Then 미연결 상담만 최신순으로 보인다.
3. Given 특정 날짜/체어로 좁히고 싶을 때, When 정렬·필터를 적용하면, Then 결과가 날짜별로 묶여 표시된다.

### US2 — 환자별 상담 카드 간략 보기 (Priority: P1)
연결된 환자를 열면 상담 카드가 길게 나열돼 보기 힘드므로, 기본은 **접힌(간략) 카드**로 보고 필요한 것만 펼친다. 환자 안에서도 키워드로 좁힌다.
**Acceptance**:
1. Given 환자 상세에 상담이 여러 건일 때, When 페이지를 열면, Then 카드가 **접힌 상태(한두 줄 미리보기+날짜)** 로 나열된다.
2. When 한 카드를 클릭하면, Then 그 카드만 본문 전체가 펼쳐진다(기존 편집·전송·연결변경 동작 유지).
3. When 환자 내 검색어를 입력하면, Then 해당 환자의 상담 중 매칭만 보인다.

### US3 — 연결완료 상담도 바로 전체복사·편집 (Priority: P2)
전체보기 목록에서 연결완료 상담도 인라인으로 전체복사·편집 접근이 가능해, 덴트웹에 빨리 붙여넣는다. (A3 잔여 — 환자상세엔 이미 있음, 전체보기/홈에서도)
**Acceptance**:
1. Given 전체보기 목록의 연결완료 상담, When 카드를 펼치면, Then 전체복사 버튼과 편집(환자상세) 접근이 보인다.

### Edge Cases
- 결과 0건 → "해당 조건의 상담이 없어요" 안내.
- 키워드는 본문 HTML이 아닌 텍스트 기준으로 매칭(태그 제외).
- 대량(수백 건) → 서버 페이징/limit으로 과다 로드 방지.
- 미연결 상담은 환자명이 없으므로 '미연결'로 표기.

## Requirements

- **FR-001**: 시스템은 기관 범위 내 상담 기록을 **키워드(본문)·연결상태(전체/연결/미연결)·체어·환자·기간·정렬(최신/오래된)** 로 조회하는 서버 측 검색을 제공해야 한다.
- **FR-002**: 검색 결과는 연결·미연결 상담을 **함께** 다루되 상태를 구분 표시해야 한다.
- **FR-003**: 전체보기 화면은 결과를 **날짜별 그룹**으로 보여주고, 카드는 **접기/펼치기**가 가능해야 한다.
- **FR-004**: 펼친 카드(연결·미연결 모두)는 **전체복사**가 가능하고, 연결완료는 편집(환자상세) 접근이 있어야 한다(A3).
- **FR-005**: 환자 상세의 상담 목록은 **기본 접힌(간략) 카드**로 나열되고 클릭 시 펼쳐지며, 기존 동작(편집·확정·전송·연결변경·삭제)은 보존되어야 한다.
- **FR-006**: 환자 상세에서 **해당 환자 내 키워드 검색**으로 카드를 좁힐 수 있어야 한다.
- **FR-007**: 홈 '기록·활동'에서 **전체보기(검색·필터 화면)** 로 진입할 수 있어야 한다.
- **FR-008**: 모든 조회는 기관 격리(institution_id)·서버 권위를 지키고, 환자 PII(주민번호 등)는 목록에 평문 노출하지 않아야 한다(이름·차트번호 수준만).
- **FR-009**: 대량 결과는 limit/페이징으로 안전 처리하고, 더 볼 수 있으면 '더 보기'를 제공해야 한다.

## Success Criteria
- **SC-001**: 수백 건이 쌓여도 키워드·필터로 원하는 상담을 몇 초 내 찾는다.
- **SC-002**: 환자 상세가 길게 줄줄이가 아니라 접힌 카드로 한눈에 들어오고, 필요한 것만 펼쳐 본다.
- **SC-003**: 연결완료 상담도 전체보기에서 바로 전체복사해 덴트웹에 붙여넣을 수 있다.
- **SC-004**: 기존 환자상세 동작(편집·전송·연결변경 등) 회귀 0.

## 설계 (Design — 코드 맵 기반, research 대체)
- **데이터**: `app/actions/consultations.ts`에 `searchConsultations(filters)` 신규 — `consultation` 기관범위 조회 + patient 조인(이름·차트), 키워드 ilike(content), 상태/체어/환자/기간 필터, 정렬, limit+offset. 반환 통합행 `{id, content, created_at, linked_at, chair_id, patient_id, patient_name, status, prescriptions, has_audio, sms_sent_at}`. 기존 `getAllUnlinkedRecords`·`getConsultationsByPatientId`는 유지.
- **전체보기 화면**: `app/(dashboard)/records/page.tsx`(신규) + `components/records/records-browser.tsx`(신규 client) — 검색창·상태/체어 필터·정렬·날짜 그룹·접이식 카드(펼치면 전체복사·편집 링크). `maxDuration`은 dashboard layout 상속.
- **환자별(A6)**: `components/consultation-history.tsx` 수정 — 카드 기본 접힘+클릭 펼침 + 환자 내 키워드 검색. 서브 컨트롤(DraftActions/Relink/ConfirmedEdit/Sms) 보존.
- **홈(A5)**: `components/home/home-feed.tsx` — '전체보기'에 `/records` 링크 추가(기존 인라인 펼침은 빠른 보기로 유지).
- **공용 헬퍼**: stripHtml/formatDate 등은 필요한 곳에 소규모로(과도 추상화 금지). CopyAllButton 재사용.
- **스키마 변경 0 · 마이그레이션 0 · 새 버킷/라이브러리 0.**

## Out of Scope
- 환자 전송 레포트 고도화·보이스피싱 UX(A7, 후속).
- 동시편집(C-05).
- 전체보기에서의 인라인 편집(편집은 환자상세로 이동) — v1은 열람·검색·복사 중심.
