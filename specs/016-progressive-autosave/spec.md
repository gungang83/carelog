# Feature Specification: 점진 청크 전사 + 상담 종료/자동저장

**Feature Branch**: `016-progressive-autosave` (작업: `claude/eo-sso-token-verify-w9v931`)
**Created**: 2026-06-29 · **Status**: Implemented
**Extends**: spec 008(상담보드) · spec 010(청크 전사)

## Summary

상담 종료 시 전사를 한참 기다리던 소모를 줄인다. 두 축:

1. **점진 청크 전사**: 청크 모드에서 구간이 잘릴 때마다(녹음 중) **즉시 백그라운드 전사**해 결과를 누적.
   종료 시엔 남은 마지막 구간 + 전체 요약만 처리 → 종료 후 대기가 "전체 길이 전사"에서
   "마지막 구간 + 요약"으로 크게 줄어듦. (기존 spec 010은 종료 후 일괄 전사였음)
2. **상담 종료 / 종료 및 저장 버튼**: "중지 및 변환" → **"상담 종료"**로 개명 + **"상담 종료 및 저장"**
   버튼 추가. 종료 후 전사가 끝나면 **자동 저장** → 사용자는 결과를 바라보며 기다리지 않아도 됨.
   실패는 비차단으로 **로그(서버 Vercel 로그) + 임시본 보존(복구 가능)**.

## User Scenarios

### US1 — 점진 전사(청크)
직원이 청크 모드로 18분 상담을 녹음한다. 녹음 중 5분 구간마다 백그라운드 전사가 돌아
("3구간 전사됨" 표시), 종료 시점엔 앞 구간이 이미 전사돼 있어 거의 바로 마무리된다.

### US2 — 종료 및 저장(자동저장)
직원이 상담을 마치며 **"상담 종료 및 저장"**을 누른다. 보드를 닫고 다음 환자로 이동해도,
백그라운드에서 전사 완료 후 자동 저장된다. 완료는 알림(spec 012)으로 확인.

### US3 — 자동저장 실패 안전망
전사·저장이 실패하면 작성 내용·녹음은 IndexedDB 임시본으로 보존되고, 서버 로그에 사유가 남으며,
사용자에겐 "보관됐어요 — 다시 저장해 주세요" 안내가 뜬다(유실 0).

## Requirements

- **FR-001** 청크 모드 녹음 중 구간 완료 시 즉시 백그라운드 전사(구간별 1회 재시도·실패 격리).
- **FR-002** 종료 시 진행 중 전사 대기 + 누락 구간 안전망 전사 → 순서대로 이어붙여 전체 요약.
- **FR-003** 녹음 바 "상담 종료" + "상담 종료 및 저장" 버튼. 후자는 체어 선택 시 활성.
- **FR-004** "상담 종료 및 저장": 종료 → 전사 완료 → 자동 `saveChairRecord` → 음성 보관 업로드(비차단).
- **FR-005** 자동저장은 보드를 닫아도(레이아웃 상시 마운트) 백그라운드로 완료.
- **FR-006** 비교 모드 + 자동저장이면 첫 결과(basic)로 자동 확정(사람 선택 불가 상황 처리).
- **FR-007** 실패(전사/저장/내용없음/체어없음) 시: 임시본 보존 + 사용자 안내 + 서버 로그(`reportAutoSaveFailure`, 환자 PII 미포함).
- **FR-008** 전사·자동저장 실패가 상담 데이터·기존 수동 저장 흐름을 막지 않음(비차단).

## Success Criteria

- **SC-001** 청크 18분 상담의 종료 후 대기가 체감상 "마지막 구간+요약" 수준으로 단축.
- **SC-002** "상담 종료 및 저장" 한 번으로 검토 없이 저장 완료(보드 닫고 이동 가능).
- **SC-003** 어떤 실패도 작성물·녹음을 잃지 않음(임시본 복구 + 로그).

## Out of Scope

- 서버 비동기 전사(탭 생명주기 완전 분리) — 후속(US: 탭 닫기·모바일 잠금에도 100% 보장).
- 실시간 스트리밍 전사(새 STT 백엔드) — 후속.
- 자동저장본 "미검토" 표식 — 현재는 저장 후 수정 가능으로 완충.

## Risks (인지)

- 자동저장은 검토 없이 저장 → 전사 오류가 그대로 기록될 수 있음(저장 후 수정으로 완충).
- 전사는 여전히 클라 탭에서 동작 → 탭 닫기·모바일 잠금 시 마지막 구간 유실 가능(점진 전사로 폭·확률 축소). 완전 보장은 서버 잡(후속).
