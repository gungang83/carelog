# Feature Specification: 일일 사용 리포트

**Feature Branch**: `014-daily-usage-report` (작업: `claude/eo-sso-token-verify-w9v931`)
**Created**: 2026-06-29 · **Status**: Implemented
**Depends on**: spec 013(사용량·크레딧) — menu_usage_daily · credit_log 데이터 출처

## Summary

하루(**KST 0~24시**) 동안의 **메뉴 사용량 + AI 토큰/크레딧 사용량**을 집계해 리포트로 발행하고,
**매일 아침 08:00(KST) 슈퍼어드민에게 알림(알림함 + 웹푸시)**으로 전달한다.

- 지금은 **전체 워크스페이스 상세 리포트**(슈퍼어드민용)를 먼저 만든다.
- 생성기는 `scope`로 일반화 — 추후 **운영자(기관 관리자)용 자기 워크스페이스 리포트**로 재사용(코드 재사용, 배선만 추가).
- 토큰: Claude 응답 `usage`(입력/출력) **실토큰**을 캡처. Whisper는 토큰 개념이 없어 전사 건수로 집계. 크레딧(기능별 단가)과 함께 표시.

## User Scenarios

### US1 — 아침 리포트 수신 (슈퍼어드민)
매일 아침, 슈퍼어드민이 알림함/푸시로 "📊 일일 사용 리포트 (날짜) · 진입 N · 크레딧 N · 전사 N건 · 토큰 N"을 받는다.
클릭하면 전일(KST 0~24시) 전체 워크스페이스 상세 리포트로 이동한다.

### US2 — 리포트 열람
리포트에서 요약(워크스페이스 수·진입·활성자·크레딧·전사·토큰, 전일 대비 ▲▼), 워크스페이스별 현황,
기능별(크레딧·토큰), 메뉴별(역할분해), 사용자별 상위, 주의 신호(잔액 음수/임박·사용 급증)를 본다.

### US3 — 과거/임의 일자 열람
`/admin/usage/report/{YYYY-MM-DD}`로 특정 일자 리포트를 본다. 발행본이 있으면 그대로, 없으면 즉석 집계.

## Requirements

- **FR-001** 매일 08:00 KST(= 0 23 * * * UTC) Vercel Cron이 전일(KST 0~24시) 리포트를 발행.
- **FR-002** 집계 범위: `menu_usage_daily`(KST 일자) + `credit_log`(KST 경계). scope='all'(전체) 또는 institution_id.
- **FR-003** 리포트 스냅샷을 `usage_reports`에 멱등 저장(과거 열람·이력). 누락 시 즉석 재집계 가능.
- **FR-004** 슈퍼어드민에게 알림함(본인 전용) + 웹푸시(본인 기기) 전달. 클릭 → 상세 리포트.
- **FR-005** Claude 응답 실토큰(입력/출력)을 `credit_log`에 기록하고 리포트에 표시. Whisper는 전사 건수.
- **FR-006** 리포트/발행/cron은 서버 권위(service_role). 열람은 슈퍼어드민 전용.
- **FR-007** cron은 CRON_SECRET Bearer(자동) 또는 슈퍼어드민 세션(수동 재발행 `?date=`)으로 인증.
- **FR-008** 환자 PII 미적재(직원 이메일·메뉴·기능·집계 수치만).

## Success Criteria

- **SC-001** 매일 아침 슈퍼어드민이 전일 전체 현황을 알림 1건 + 1클릭으로 파악.
- **SC-002** "누가·얼마나·어떤 기능·몇 토큰"이 일자 단위로 분해됨.
- **SC-003** 동일 일자 재발행이 멱등(중복 리포트/알림 폭주 없음).

## Out of Scope

- 실결제·청구서. 이메일/SMS 채널(후속 — 운영자 배포 시).
- 운영자(기관별) 리포트 자동 발송(생성기는 지원, 배선은 후속).
- 시간대 사용자 설정(현재 KST 고정).
