# Quickstart: 실시간 체어 상담기록 알림 검증

## 선행(배포·설정)

1. **마이그레이션 적용**: `chair_audit_logs`를 `supabase_realtime` publication에 추가하는 마이그레이션을 Supabase에 적용.
   - 확인: Supabase 대시보드 → Database → Replication에서 `chair_audit_logs`가 realtime에 포함되는지.
2. Vercel 프로덕션/프리뷰 배포(기존 env 그대로, 신규 env 없음).

## US1 — 실시간 토스트 + 목록 자동 갱신 (P1)

1. 같은 기관 계정으로 **기기 A(데스크)**·**기기 B(2번 체어)** 두 화면에서 홈을 연다.
2. 기기 B에서 2번 체어로 상담을 녹음·저장(`saveChairRecord`).
3. **기기 A**: 5초 이내 "2번 체어 · 상담 기록 도착" 토스트가 뜨고, 미연결 기록 목록에 새 카드가 **수동 새로고침 없이** 추가됨 ✅
4. **기기 B(올린 본인)**: 토스트·소리 없이 목록만 갱신(에코 방지) ✅
5. 타 기관 계정 화면: 아무 알림 없음 ✅

## US2 — 소리 (P2)

1. 기기 A에서 "🔔 알림 소리 켜기" 1회 탭.
2. 기기 B에서 새 기록 저장 → 기기 A에서 효과음 재생 ✅
3. 소리 켜기 전 상태: 시각 알림만, 무음, 콘솔 에러 없음 ✅
4. 연속 저장 3건 → 효과음이 겹쳐 시끄럽지 않게 디바운스 ✅
5. 소리 끄기 토글 후: 시각 알림 유지·무음 ✅

## US3 — 백그라운드 Web Push (P3)

1. 기기(안드로이드 폰)에서 푸시 허용, Carelog를 백그라운드로.
2. 다른 기기에서 체어 기록 저장 → OS 푸시 "새 상담 기록 / {체어} 상담 기록이 올라왔습니다" 도착, 탭하면 홈으로 이동 ✅
3. 포그라운드 기기: 인앱 토스트가 주, 푸시 과중복 없음 ✅

## 재연결 (FR-011)

1. 기기 A 네트워크 잠시 차단 → 그 사이 기기 B에서 기록 저장 → 기기 A 재연결 시 목록이 최신으로 재동기화 ✅

## 빌드/품질
