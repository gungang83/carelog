# spec 027 (기획) — 상담 세션 지속성·안전망

> 상태: **기획 — 대표님 논점 확정 대기.** 7/8 김도은 회의 '기능 검토' 6건의 명세화.
> 작성: 다온 · 2026-07-08 · 코드 근거: chair-provider.tsx(레이아웃 레벨 마운트), chair-overlay.tsx, consultation-board.tsx

## 0. 현행 구조 핵심 사실

녹음 상태는 화면(오버레이/보드)이 아니라 **대시보드 레이아웃 레벨의 ChairProvider**에 산다.
- ✅ 오버레이 X를 눌러도 **녹음은 이미 계속된다**(CLOSE_OVERLAY는 UI만 닫음). 앱 내 다른 페이지로 이동해도 유지.
- ⚠️ 그러나 **사용자가 그 사실을 알 수 없다** — 닫고 나면 어디에도 "녹음 중" 표시가 없음(홈 체어 카드 제외).
- ⚠️ 탭 닫기/새로고침은 녹음 유실(MediaRecorder 소멸). beforeunload 경고는 **보드 컴포넌트에 등록**되어 있어, 보드가 언마운트된 상태(닫고 다른 페이지)에서 녹음 중이면 **경고 없이 탭을 닫을 수 있는 구멍**이 있다.

→ 회의 질문 "X 눌러도 상담 계속 진행??"의 답은 "이미 그렇다". 이번 스펙의 본질은 **그 지속성을 보이게 하고(상시 표시), 구멍을 막고(전역 이탈 경고), 방치를 구조하는(타이머)** 것.

## 1. 항목별 기획

### ① 상시 '상담중(녹음중)' 플로팅 표시 ★핵심
- ChairProvider가 렌더하는 **전역 플로팅 필**(레이아웃 레벨 — 어느 페이지에서든 보임):
  `🔴 상담중 (녹음중) · [체어명] · 12:34` — 탭하면 해당 체어 오버레이 재오픈.
- 녹음/일시정지 중인 체어가 1개 이상이면 표시. 복수면 개수 뱃지.
- **beforeunload 경고를 Provider로 승격** — 보드가 닫혀 있어도 녹음 중이면 탭 닫기/새로고침 경고(위 구멍 봉합).

### ② 방치 대비 — '상담 계속하기' 타이머 (2026-07-08 대표 확정 반영)

**확정**: 방치 기준 10분 · 무응답 시 **자동 '종료 및 저장'** · 응답 대기 시간 **설정 가능**.

#### ②-1. '방치'의 판단 — 활동 신호 2종 (대표 질문 회신)
"마우스 움직임 없음"만으로는 부족하다 — **치과 상담은 손을 안 대고 말만 하는 시간이 길다**(체어에서 태블릿 내려놓고 대화). 마우스 기준만 쓰면 대화 중인 상담을 방치로 오판한다. 그래서 활동 = 둘 중 하나:

| 신호 | 감지 방법 | 의미 |
|---|---|---|
| **조작** | pointermove·pointerdown·keydown·touchstart·scroll (스로틀 수 초 단위) | 사람이 화면을 만지고 있음 |
| **음성** | 이미 녹음 중인 마이크 스트림에 AnalyserNode를 붙여 입력 레벨(RMS)이 임계 이상이면 '대화 중' | **상담이 진행되고 있음** — 조작이 없어도 방치 아님 |

- 활동(둘 중 아무거나) 발생 → 방치 타이머 **리셋**.
- 음성 감지는 배경 소음 오탐 가능성이 있어 임계값 보수적으로 + 기관 설정에서 on/off 가능(기본 on).
  ※ 추가 다운로드·서버 비용 0 — 이미 열려 있는 마이크 스트림을 읽기만 한다.

#### ②-2. 흐름
```
녹음 중 무활동 10분(idle, 설정 가능)
  → 경고 상태: 플로팅 필 amber 펄스 + 모달 "상담이 계속 녹음되고 있어요. 계속하시겠어요?"
     [계속하기] [종료 및 저장] + 알림음 + 웹푸시(본인 기기 — 자리 이탈 대비)
  → 응답 대기 M분(grace, 설정 가능, 기본 5분) 카운트다운 표시
  → 무응답 → 자동 '종료 및 저장' 실행(서버 비동기 전사 경로 — 3분↑ 녹음 자동 서버 위임이라 안전)
     + 알림함 통지 "3번 체어 상담이 방치되어 자동 저장되었습니다"
```
- 자동 저장이지 **폐기가 아니다** — 잘못 종료됐어도 기록은 남고, '이어서 상담'(④)으로 복구 가능.
- **설정**(owner/admin, /settings 상담 안전망): idle 분(기본 10) · grace 분(기본 5) · 음성 감지 on/off.
  저장: `institutions.consult_settings jsonb` (ALTER 1컬럼 — 소테이블 불필요, 후속 설정 확장 수용).

#### ②-3. 플로팅의 기술적 한계와 답 — "다른 프로그램 위" (대표 질문 회신)
Carelog는 웹앱(브라우저/홈화면 설치 PWA)이라 **OS 권한이 없어 임의로 다른 프로그램(차트 등) 위에 UI를 얹을 수 없다.** 이건 웹 샌드박스의 근본 제약. 단, 예외 통로가 셋 있고 조합하면 실질적으로 커버된다:

| 통로 | 가능한 것 | 지원 |
|---|---|---|
| **Document Picture-in-Picture** | **항상-위(always-on-top) 미니창**에 우리 HTML을 띄움 — 덴트웹 등 다른 프로그램 위에 "🔴 상담중 · 3번 체어 · 12:34 · 무활동 4분" 소형 창 상주 | 데스크톱 Chrome/Edge ✅ (상담실 PC 시나리오) · iPad/모바일 ❌ |
| **탭 제목·파비콘** | 브라우저 탭 제목을 "🔴 녹음중 12:34 — Carelog"로, 파비콘에 빨간 점 — 작업표시줄·탭 목록에서 인지 | 전 플랫폼 ✅ |
| **웹 푸시(OS 알림)** | 경고 시점(②-2)에 OS 알림 — 다른 프로그램 사용 중에도 뜸 | 기존 push_subscriptions 인프라 재사용 ✅ |

- 설계: **인앱 플로팅 필 = 기본**(모든 기기) + **"항상 위 미니창" 버튼**(지원 브라우저에서만 노출 — Document PiP로 승격) + 탭 제목 + 경고 시 푸시. iPad는 인앱 필+푸시로 커버.
- 필/미니창 표시 내용: 상태(녹음중/기록중) · 체어명 · 경과 시간 · **무활동 경과**(활동 시 리셋, 2분 이상부터 노출 "마지막 활동 4분 전") · 탭하면 상담 화면 복귀.

### ③ '녹음 없이 상담' 옵션
- 현행: 상담 시작 = 즉시 녹음. 제안: 시작 지점(히어로·체어)에 **"녹음 없이 기록만"** 보조 진입.
- 동작: 마이크 없이 보드/오버레이 오픈 — 에디터·참여자·자료·견적 전부 동일, 저장 경로 동일. 상태 `note` (idle/recording/paused와 구분, 상시 필에는 "상담중 (기록)"로 표시 — 방치 타이머 제외).
- 도중 "지금부터 녹음" 전환 버튼(가능하면 v1, 복잡하면 후속).

### ④ 기존 상담에 '이어서 상담'
- 진입: 상담 카드(연결/미연결 공용 카드)에 **"이어서 상담"** 액션.
- 동작: 해당 기록을 에디터에 로드한 채 보드 오픈 → 본문 끝에 구분 블록 자동 삽입:
  `[이어서 상담 — 7/10 14:20 · 상담: 송정현]` → 이후 녹음 요약·메모가 아래로 append → 저장 시 **같은 consultation 업데이트**.
- 상담자 교체 기록: v1은 구분 블록의 상담자 표기 + participants 스냅샷 갱신(**스키마 변경 없음**).
  author 이력 배열(정식 데이터화)은 상담 데이터화 스펙에서.
- 주의: 다른 체어에서 이어서 열 때 체어 상태 충돌 → 이어서 상담은 **체어 무관 모드**(chair_id 유지, 녹음 세션만 새로)로 단순화.

### ⑤ 에디터 '저장 위치' 안내
- 보드/오버레이 에디터 상단 한 줄: `이 내용은 [체어명] 상담 기록으로 저장됩니다` (이어서 모드면 `기존 기록(환자명/날짜)에 이어서 저장됩니다`).
- 구현 미미(안내줄) — ④와 함께.

### ⑥ 녹음 화면(오버레이) → 상담보드 확장 진입
- 오버레이 헤더에 **"⛶ 보드로 확장"** — 상태는 Provider 공유라 그대로 전체 보드 화면 전환(녹음 연속).
- 반대(보드→간이 오버레이)는 불필요(닫기+플로팅 필로 충분).
