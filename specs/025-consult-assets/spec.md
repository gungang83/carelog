# spec 025 (기획 v2) — 상담 이미지 라이브러리 + 상담 캔버스

> 상태: **기획 v2 — 대표님 1차 피드백 반영(세션 66).** 구현 착수 전 최종 확인 대기.
> 작성: 다온 · 2026-07-08

## 1. 대표님 피드백으로 확정된 방향 (v1 대비 변경)

1. **"템플릿 먼저 고르고 시작" 개념 폐기.** 상담은 흐름에 따라 이미지를 **계속 추가**하는 작업 — 프리셋을 깔고 들어가는 구조가 아님. → 핵심은 **상담 중 언제든 여는 자료 픽커**. (묶음 프리셋은 사용 패턴을 본 뒤 후속 검토)
2. **에디터를 "상담 캔버스"로 강화.** 상담하면서 크게 쓰고, 이미지·글 배치가 자유로워야 함. → v1 축으로 승격.
3. **Carelog 제공(전역) 콘텐츠는 v1 제외.** 예미안 자산 업로드부터 테스트. 스키마만 전역 확장 가능하게 열어둔다(institution_id nullable).

## 2. v1 범위 — 두 축

### 축 A. 기관 이미지 라이브러리
- `consult_assets`: id · institution_id(nullable, **v1은 항상 기관값** — null=전역은 후속) · title · category · image_url · caption(선택) · display_order · active · created_by · created_at.
- RLS: read=자기 기관 멤버십(+후속: 전역 active), write=owner/admin 멤버십. 스토리지 `consult-assets` 버킷 `<institution_id>/` 폴더, 기존 webp 압축·optimize·lazy 파이프라인 재사용.
- 관리: `/settings` "상담 자료" 섹션 — 업로드(제목·카테고리·캡션)·활성 토글·순서·삭제. 카테고리는 lib config 확장형(임플란트·신경치료·보철·교정·치주·예방·장치·기타).
- **에디터 픽커**: 공용 `rich-text-editor` 툴바에 "📚 자료" — 카테고리 필터+검색, 썸네일 그리드, **클릭 확대 미리보기**, 선택 시 커서 위치 삽입(캡션 동반 선택). 상담 중 몇 번이고 열 수 있음. 픽커에서 즉석 업로드(새 자료 등록)도 지원 — 상담 중 찍은 사진을 라이브러리에 올리며 바로 삽입.

### 축 B. 상담 캔버스 (에디터 강화)
현재 에디터(TipTap)에 이미 있는 것: 이미지 리사이즈 핸들 · 붙여넣기/드롭 삽입 · 주석(annotator) · 서식/목록/제목. v1에서 추가:
1. **전체화면 편집 모드** — 상담 동안 화면 가득(토글 ⛶). 체어 에디터·카드 편집·폼 공통.
2. **이미지 정렬·글 감싸기** — 왼쪽/가운데/오른쪽 + 텍스트 랩(float). 이미지 옆에 설명을 쓰는 배치.
3. **이미지 나란히(2장 가로)** — 전후 비교·단계 비교. (이미지 2장 선택 → 나란히 묶기)
4. 픽커(축 A)와 한 툴바에 통합.

**배치 자유도의 경계(중요)**: "자유 배치"를 절대좌표 캔버스(그림판/피그마식)로 가면 환자 포털 표시·모바일 반응형·전체 복사(평문)와 충돌한다. v1은 **HTML 플로우 안에서의 자유도**(크기·정렬·감싸기·나란히)로 구현하고, 진짜 프리폼 캔버스가 필요하면 "상담 보드" 별도 화면으로 후속 검토. → 논점 ①

## 3. 사용 시나리오 (예미안 테스트 기준)

1. 사전: 실장/원장이 `/settings`에 임플란트 단계 그림·장치 사진 등 업로드(카테고리 지정).
2. 상담 시작(체어 즉시 기록 or 녹음): 에디터 전체화면 → "📚 자료"에서 치식도 삽입 → 환자와 보며 설명, 필요하면 리사이즈·주석 → 상담 진행하며 이미지 추가·글 작성 반복.
3. 종료: 저장하면 기존 상담 기록과 동일하게 카드·환자 포털에 표시(이미지 인라인). 녹음 요약과 공존.

## 4. 비범위 (후속)

- 전역(Carelog 제공) 자산 + `/admin/assets` 발행 UI — 스키마는 준비됨, UI만 후속.
- 묶음 프리셋(자주 쓰는 조합 저장) — 사용 패턴 확인 후.
- 프리폼 캔버스(절대 배치), 프레젠테이션 전용 뷰어, 자산 사용 통계, 원내 전용(포털 비노출) 플래그, 동영상.

## 5. 남은 논점

| # | 논점 | 다온 의견 |
|---|---|---|
| ① | 배치 자유도 수준 | **HTML 플로우 내 자유(크기·정렬·감싸기·나란히)로 v1.** 절대좌표 캔버스는 포털·모바일·복사와 충돌 — 필요성 확인 후 별도 화면으로 |

## 6. 구현 순서 (착수 시)

1. 마이그레이션 `consult_assets` + 버킷 → 2. `lib/consult-assets.ts` + 액션 → 3. `/settings` 관리 섹션 → 4. 에디터 픽커 → 5. 에디터 강화(전체화면 → 정렬/감싸기 → 나란히) → 6. 문서·업데이트 피드.
