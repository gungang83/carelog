# spec 028 (기획) — 치료비 견적 빌더

> 상태: **기획 — 대표님 논점 확정 대기.** (spec 026 P3-B에서 분리, 2026-07-08 대표 지시)
> 작성: 다온 · 2026-07-08 · spec 027은 상담 세션 지속성으로 예약 유지.

## 1. 목표

상담 중 치료비 견적을 **몇 번의 클릭으로** 만들어 상담 기록에 깔끔하게 삽입하고, 그대로 환자에게 전달한다.
"얼마라고 했더라?"를 없애고, 견적 커뮤니케이션을 상담 기록의 일부로 만든다.

## 2. 두 구성 요소

### A. 치료 항목 사전 (기관 프리셋)
- `/settings`에 **"치료 항목·수가"** 섹션(owner/admin): 항목명 · 기본 단가 · 표시 순서 · 활성.
  예: 임플란트(뼈이식 포함) 1,500,000 / 지르코니아 크라운 500,000 / 라미네이트 700,000 / 스케일링(보험) …
- 테이블 `treatment_items`: id · institution_id · name · price(integer, 원) · display_order · active · created_at.
  RLS: read=멤버십 / write=owner·admin (consult_assets와 동일 패턴). 마이그레이션 1건.
- 프리셋은 **참고 단가** — 빌더에서 건별 수정 가능(케이스마다 다름).

### B. 견적 빌더 (에디터 팝업)
- 에디터 툴바 **"₩ 견적"** 버튼 → 팝업:
  1. 프리셋 검색/클릭 → 행 추가(이름·단가 자동 채움)
  2. 행 편집: 수량 · 단가 · 직접 입력 행(프리셋에 없는 항목) · 행 삭제
  3. **합계 자동 계산** (할인은 음수 금액 행으로 — 별도 개념 없이 단순하게)
  4. 메모 한 줄(선택): 예 "보험 적용 여부에 따라 달라질 수 있습니다"
  5. [기록에 삽입]
- 터치 친화(체어 태블릿): 큰 행 높이·스텝퍼(수량 ±)·숫자 키패드(inputMode).

## 3. 삽입 형식 — 표가 아니라 "정돈된 평문 블록" ★핵심 결정

에디터(TipTap StarterKit)는 표를 지원하지 않고, 표 확장을 붙여도 **차트 '전체 복사'(평문)와 충돌**한다.
상담 요약 평문화(세션 66)와 같은 문법인 **[라벨] 평문 블록**으로 삽입한다:

```
[치료비 견적]
- 임플란트(뼈이식 포함) 2개 × 1,500,000원 = 3,000,000원
- 지르코니아 크라운 1개 × 500,000원 = 500,000원
- 상담 할인 = -200,000원
합계: 3,300,000원
※ 보험 적용 여부에 따라 달라질 수 있습니다
```

- 화면(카드·환자 포털)에서 그대로 읽기 좋고, **차트에 복사해도 형식이 유지**된다.
- 형식을 고정해 두면 후속 '상담 데이터화'(roadmap 다음 항목 — outcome·금액 필드) 때 **파싱 가능** —
  지금은 본문이 곧 데이터, 구조화 저장(테이블화)은 그 스펙에서 결정.
- 수량 1이면 "× 1개" 생략, 금액은 천 단위 쉼표.

## 4. 범위 (v1)

포함: 치료 항목 사전(설정) · 빌더 팝업(프리셋+직접 입력+합계+메모) · [치료비 견적] 블록 삽입.
제외(후속):
- 견적 DB 저장·이력·버전 비교 — '상담 데이터화' 스펙과 함께(형식 파싱으로 소급 가능).
- 상담→수납 전환 분석, EO 회계 연동(수납은 EO 영역 — Carelog는 "상담에서 설명한 금액의 기록"까지).
- 보험/비보험 구분 필드, 부가세 — 논점 ② 결과에 따라.

## 5. 구현 스케치 (확정 후)

1. 마이그레이션 `treatment_items` → 2. `lib/treatment-items.ts` + `app/actions/treatment-items.ts`(CRUD, owner·admin 가드) → 3. `/settings` 섹션 → 4. `components/estimate/estimate-builder.tsx`(팝업) + 에디터 툴바 배선 → 5. 문서·업데이트 피드.
- 견적 블록 생성은 순수 클라 로직(문자열 조립) — 서버 불필요. DB는 프리셋만.
