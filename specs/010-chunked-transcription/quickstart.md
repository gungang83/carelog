# Quickstart: 긴 상담 청크 분할 전사 검증

> 대상: 예미안(lab_enabled) 워크스페이스, 프로덕션 또는 preview. 목적: 통짜 vs 청크 A/B 검증(C안).

## 사전 조건
- 워크스페이스 `lab_enabled = true`(예미안).
- 음성 전사 키(OpenAI/Anthropic) 설정됨.
- spec 009 음성 보관 버킷·정책 적용 상태(재청취 검증 시).

## 시나리오 1 — 긴 상담 완료 (SC-001)
1. 홈 히어로에서 **녹음 엔진 → "긴 상담"** 선택.
2. "상담 기록 시작" → **30~40분** 녹음(테스트는 음원 재생으로 대체 가능).
3. 종료 → "n/m 구간 전사 중" 진행률 노출 확인.
4. **기대**: 페이지 크래시/전손 없이 전 구간 전사문 + 전체 요약이 본문에 삽입, 저장 가능.

## 시나리오 2 — 실패 격리 (SC-002)
1. 긴 상담 모드로 녹음·종료.
2. (개발 검증) 한 구간 전사를 강제 실패시키거나 네트워크 일시 차단.
3. **기대**: 실패 구간만 표시/재시도되고, 성공 구간 전사문은 보존되어 본문에 들어옴. 전체 전손 0.

## 시나리오 3 — 통짜 vs 청크 비교 (SC-003·SC-004)
1. 같은 음원을 **기본모델**로 1회, **긴 상담**으로 1회 전사.
2. **기대**: 두 결과를 나란히 비교 — 전사 정확도, 요약 전체 맥락(증상→처치→처방 흐름 보존), 체감 속도. 청크 요약 맥락이 통짜 대비 동등 이상.

## 시나리오 4 — 짧은 녹음 (Edge)
1. 긴 상담 모드로 **3분** 녹음·종료.
2. **기대**: 구간 1개로 처리되어 통짜와 동등하게 정상 동작.

## 시나리오 5 — 크래시 복구 (FR-011)
1. 긴 상담 녹음 종료 후 전사 중 탭 새로고침/강제 종료.
2. 재진입 → 보드 열기.
3. **기대**: 복구 배너 → 복구 시 구간 녹음물로 **재전사** 진행, 본문 복구.

## 시나리오 6 — 비-lab 회귀 (SC-006)
1. lab 아닌 워크스페이스로 로그인.
2. **기대**: 녹음 엔진 픽커 미노출(또는 chunk 선택 불가), 기본(basic)으로 동작. 동작·결과가 도입 전과 동일.

## 빌드/배포 체크
- `npm run build` 그린(TypeScript·compile).
- 스키마 변경 없음 → 마이그레이션/버킷 추가 작업 없음.
- 배포는 다온 마무리 프로토콜(dev→main).
