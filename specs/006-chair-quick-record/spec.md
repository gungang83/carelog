# Feature Specification: Chair Quick Record

**Feature Branch**: `006-chair-quick-record`  
**Created**: 2026-05-25  
**Status**: Draft  
**Input**: 체어 즉시 녹음 오버레이 — 치과 직원이 환자 선택 없이 즉시 상담 내용을 녹음·기록할 수 있는 글로벌 오버레이 기능

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 즉시 녹음 시작 (Priority: P1)

치과 직원이 환자를 체어에 앉히는 순간 헤더의 체어 버튼(A/B/C)을 탭하면, 현재 보고 있는 화면을 벗어나지 않고 오버레이가 열리며 즉시 녹음을 시작할 수 있다. 녹음 중에 오버레이를 닫아도 녹음은 계속 진행된다. 오버레이를 다시 열면 녹음 중 상태로 복귀한다.

**Why this priority**: 현재 워크플로우의 핵심 병목(환자 선택 → 화면 이동 → 녹음)을 제거하는 가장 중요한 기능이다. 이 스토리만으로도 임상 현장에서 즉각적인 가치를 제공한다.

**Independent Test**: 헤더에서 체어 A 버튼을 탭하고 오버레이에서 녹음 시작 후 오버레이를 닫은 뒤 다시 열었을 때 녹음이 계속 진행 중임을 확인한다. 녹음 종료 시 기록이 임시 저장된다.

**Acceptance Scenarios**:

1. **Given** 직원이 임의의 화면을 보고 있을 때, **When** 헤더의 체어 A 버튼을 탭하면, **Then** 현재 화면 위에 오버레이가 열리고 해당 체어의 녹음 컨트롤이 표시된다
2. **Given** 오버레이가 열려 있을 때, **When** 녹음 시작 버튼을 탭하면, **Then** 녹음이 시작되고 체어 버튼 상태가 "녹음중"으로 바뀐다
3. **Given** 녹음이 진행 중일 때, **When** 오버레이의 닫기 버튼을 탭하면, **Then** 오버레이는 닫히고 녹음은 계속 진행되며 헤더 체어 버튼에 녹음중 표시가 유지된다
4. **Given** 녹음 중 오버레이가 닫혀 있을 때, **When** 같은 체어 버튼을 다시 탭하면, **Then** 오버레이가 다시 열리고 녹음 중 상태로 복귀한다
5. **Given** 녹음이 완료되어 종료하면, **When** 시스템이 처리를 마치면, **Then** 기록이 임시 저장되고 체어 버튼 상태가 "미연결 기록 있음"으로 변경된다

---

### User Story 2 - AI 변환 및 기록 편집 (Priority: P2)

녹음이 종료되면 시스템이 자동으로 음성을 텍스트로 변환한다. 직원은 변환된 텍스트를 오버레이 내에서 확인하고 수정할 수 있으며, 최종 내용을 임시 기록으로 저장한다.

**Why this priority**: 녹음만으로는 활용도가 제한적이다. 텍스트 변환과 편집 기능이 있어야 상담 기록으로서 실질적 가치를 갖는다.

**Independent Test**: 녹음 종료 후 오버레이에 변환 대기 표시가 나타나고, 일정 시간 후 텍스트가 표시된다. 텍스트를 수정 후 저장하면 임시 기록에 반영된다.

**Acceptance Scenarios**:

1. **Given** 녹음이 종료되었을 때, **When** 시스템이 변환 작업을 시작하면, **Then** 오버레이에 변환 진행 중 상태가 표시된다
2. **Given** 변환이 완료되었을 때, **When** 직원이 오버레이를 열면, **Then** 변환된 텍스트가 편집 가능한 상태로 표시된다
3. **Given** 변환된 텍스트가 표시될 때, **When** 직원이 텍스트를 수정하고 저장하면, **Then** 수정된 내용이 임시 기록에 반영된다
4. **Given** 변환이 실패하였을 때, **When** 오류가 발생하면, **Then** 오류 안내와 함께 직접 텍스트 입력 옵션이 제공된다

---

### User Story 3 - 임시 기록에 환자 연결 (Priority: P3)

체어에 저장된 임시 기록을 실제 환자와 연결하여 정식 상담 기록으로 전환한다. 직원은 오버레이 내에서 환자를 검색하고 선택하면 임시 기록이 해당 환자의 정식 상담 기록으로 이동한다.

**Why this priority**: 즉시 녹음(P1)과 텍스트 변환(P2)으로 일단 기록을 캡처한 뒤, 진료 흐름이 안정되면 환자를 연결하는 흐름이다. P1/P2 없이는 이 기능이 의미 없다.

**Independent Test**: 미연결 기록이 있는 체어 버튼 탭 → 임시 기록 목록 확인 → 환자 검색 후 연결 → 정식 상담 기록 목록에 나타나는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 미연결 임시 기록이 있는 체어일 때, **When** 헤더의 해당 체어 버튼을 탭하면, **Then** 오버레이에 임시 기록 목록이 표시된다
2. **Given** 임시 기록 목록이 표시될 때, **When** 특정 기록에서 "환자 연결" 버튼을 탭하면, **Then** 환자 검색 화면이 오버레이 내에 표시된다
3. **Given** 환자 검색 화면에서, **When** 환자를 검색하고 선택하면, **Then** 해당 임시 기록이 선택된 환자의 정식 상담 기록으로 전환되고 체어에서 제거된다
4. **Given** 환자 연결이 완료되었을 때, **When** 더 이상 미연결 기록이 없으면, **Then** 체어 버튼 상태가 "대기"로 돌아간다
5. **Given** 임시 기록 연결 전에, **When** 직원이 기록을 삭제하려 하면, **Then** 삭제 확인 후 영구 삭제되고 감사 로그에 삭제 이벤트가 기록된다

---

### User Story 4 - 감사 추적 및 기록 보안 (Priority: P4)

모든 체어 기록 생성, 수정, 환자 연결, 삭제 이벤트가 변경 불가 감사 로그에 남는다. 로그에는 행위자 계정, 체어, 타임스탬프, 이벤트 유형이 포함된다.

**Why this priority**: 의료 기관의 책임 추적을 위한 필수 요소이나 핵심 사용 흐름이 완성된 뒤 검증 가능하다.

**Independent Test**: 체어 기록을 생성/수정/연결/삭제하는 각 행위 후 관리자 패널에서 해당 이벤트의 감사 로그를 확인한다.

**Acceptance Scenarios**:

1. **Given** 직원이 체어에 기록을 생성했을 때, **When** 관리자가 감사 로그를 조회하면, **Then** 생성 이벤트에 작성자 계정, 체어 ID, 정확한 타임스탬프가 기록되어 있다
2. **Given** 직원이 임시 기록을 환자와 연결했을 때, **When** 감사 로그를 조회하면, **Then** 연결 이벤트에 연결한 계정, 체어, 연결된 환자, 타임스탬프가 기록되어 있다
3. **Given** 감사 로그가 생성된 후, **When** 어떤 수단으로도 수정 시도가 이루어지면, **Then** 수정이 거부되고 로그 무결성이 유지된다

---

### Edge Cases

- 녹음 중에 다른 체어 버튼을 탭하면? → 현재 녹음 중인 체어 상태를 유지하고 새 체어의 오버레이를 별도로 열 수 있다 (동시 2개 체어 녹음 가능)
- 마이크 권한이 없는 경우? → 오버레이에서 권한 요청 안내를 표시하고, 거부 시 텍스트 직접 입력 모드로 대체한다
- 오버레이 열린 상태에서 브라우저/앱 탭이 닫히면? → 진행 중 녹음 데이터는 소실될 수 있으며, 저장된 임시 기록은 유지된다
- 같은 체어에 여러 임시 기록이 쌓인 경우? → 생성 시간 역순으로 목록 표시, 각각 독립적으로 환자 연결 가능
- 환자 연결 대상 환자가 다른 기관 소속인 경우? → 현재 기관의 환자 목록만 검색 범위에 포함
- AI 변환 시간이 너무 길어지는 경우? → 최대 대기 시간(2분) 초과 시 오류 안내와 재시도 옵션 제공
- 기록 없이 체어 버튼만 탭한 경우? → 오버레이가 열리고 녹음 시작 버튼만 표시되는 기본 상태

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 직원 화면 헤더에 기관에서 설정한 체어 목록(A/B/C 등)을 버튼으로 표시해야 한다
- **FR-002**: 각 체어 버튼은 현재 상태(대기 / 녹음중 / 미연결 기록 있음 / 변환중)를 시각적으로 구분하여 표시해야 한다
- **FR-003**: 체어 버튼 탭 시 현재 화면을 벗어나지 않고 오버레이 다이얼로그가 열려야 한다
- **FR-004**: 오버레이 내에서 녹음 시작, 일시정지, 종료가 가능해야 한다
- **FR-005**: 오버레이를 닫아도 진행 중인 녹음은 중단되지 않아야 한다
- **FR-006**: 녹음 종료 후 시스템은 자동으로 음성을 텍스트로 변환해야 한다
- **FR-007**: 변환된 텍스트는 오버레이 내에서 편집 가능해야 한다
- **FR-008**: 임시 기록은 patient_id 없이 저장되어야 하며, 체어 ID·작성자 계정·생성 타임스탬프가 반드시 포함되어야 한다
- **FR-009**: 직원은 임시 기록에서 환자 검색 후 연결하여 정식 상담 기록으로 전환할 수 있어야 한다
- **FR-010**: 환자 연결 시 연결한 계정·타임스탬프가 기록에 불변 필드로 저장되어야 한다
- **FR-011**: 모든 기록 생성·수정·연결·삭제 이벤트는 변경 불가 감사 로그에 자동 기록되어야 한다
- **FR-012**: 체어 기능은 직원(스태프) 계정에만 노출되며 환자 포털에는 표시되지 않아야 한다
- **FR-013**: 기관별로 체어를 설정(추가·이름 변경·비활성화)할 수 있어야 한다
- **FR-014**: 마이크 접근이 불가한 환경에서는 텍스트 직접 입력으로 대체 가능해야 한다

### Key Entities

- **Chair (체어)**: 기관 내 진료 공간 단위. 이름(A/B/C 등), 소속 기관, 활성화 여부를 갖는다
- **Chair Quick Record (체어 임시 기록)**: patient_id 없이 생성되는 상담 기록. 체어 ID, 작성자 계정, 생성 시각, 녹음 파일, 변환 텍스트, 상태(임시/연결됨/삭제됨)를 갖는다
- **Consultation (정식 상담 기록)**: 기존 엔티티. patient_id가 nullable로 변경되고 chair_id·linked_at·linked_by 필드가 추가된다
- **Audit Log (감사 로그)**: 체어 기록 관련 이벤트 로그. 이벤트 유형, 행위자 계정, 체어 ID, 대상 기록 ID, 타임스탬프를 갖는다. 불변(삽입 전용)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 직원이 체어 버튼 탭부터 녹음 시작까지 3번 이하의 탭으로 완료할 수 있다
- **SC-002**: 오버레이 열림 응답 시간이 사용자가 인지할 수 없는 수준(즉각적)이어야 한다
- **SC-003**: 녹음 종료 후 AI 텍스트 변환이 2분 이내에 완료된다
- **SC-004**: 임시 기록에서 환자 연결까지의 흐름이 5단계 이하의 조작으로 완료된다
- **SC-005**: 모든 체어 기록 이벤트의 100%가 감사 로그에 기록된다
- **SC-006**: 녹음 중 오버레이 닫기/열기를 반복해도 녹음 데이터 손실이 0%이다
- **SC-007**: 체어 버튼 상태가 실제 상태와 항상 일치하여 직원이 현황을 한눈에 파악할 수 있다

## Assumptions

- 체어 수는 기관당 최대 10개로 가정한다 (헤더 공간 제약)
- 음성-텍스트 변환은 기존 프로젝트에서 사용 중인 AI 변환 서비스를 재활용한다
- 동시에 여러 체어에서 녹음 중인 경우를 지원한다 (각 체어 독립 관리)
- 임시 기록의 보관 기간은 생성 후 30일로 가정하며, 이후 자동 삭제 및 감사 로그 보존
- 체어 설정(추가/수정)은 기관 관리자 또는 최고 관리자만 가능하다
- 환자 연결은 현재 직원의 소속 기관 환자 목록 내에서만 가능하다
- 오버레이는 모바일 브라우저(PWA)와 데스크탑 브라우저 모두에서 동작해야 한다
- 기존 consultations 테이블의 patient_id 컬럼이 nullable로 변경될 때 기존 데이터에 영향이 없다
