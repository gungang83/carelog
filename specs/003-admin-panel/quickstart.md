# Quickstart: 어드민 패널 테스트 시나리오

**Date**: 2026-05-14

---

## 사전 준비

- 계정 A: `mobys0416@gmail.com` (슈퍼 어드민)
- 계정 B: 일반 owner 계정 (기관 X 소속)
- 계정 C: 일반 staff 계정 (기관 X 소속)
- 계정 D: 계정 B와 동일 기관 X + 기관 Y 모두 소속 (복수 기관 테스트용)

---

## 시나리오 1: 단일 기관 직원 — 헤더 정상 표시

1. 계정 C로 로그인
2. 헤더 확인 → 기관명만 텍스트로 표시 (드롭다운 없음) ✅
3. `/settings` 접근 시도 → "권한이 없습니다" (staff는 설정 불가) ✅

---

## 시나리오 2: 복수 기관 직원 — 기관 전환

1. 계정 D로 로그인
2. 헤더의 기관명 클릭 → 드롭다운에 기관 X, 기관 Y 표시 ✅
3. 기관 Y 선택 → 대시보드가 기관 Y 환자 목록으로 전환 ✅
4. 새로고침 → 기관 Y 유지 ✅
5. 기관 X 재선택 → 기관 X 데이터로 전환 ✅

---

## 시나리오 3: 기관 관리자 — 직원 목록 조회

1. 계정 B (owner)로 로그인
2. 헤더의 "설정" 링크 클릭 → `/settings` 진입 ✅
3. 직원 목록에 계정 C 표시 (이름/이메일/역할/활성 상태) ✅

---

## 시나리오 4: 기관 관리자 — 직원 비활성화

1. 계정 B로 `/settings` 접속
2. 계정 C의 토글 → 비활성화 확인 팝업 → 확인
3. 목록에서 계정 C 상태 "비활성" 표시 ✅
4. 계정 C로 로그인 → 기관 X 대시보드 접근 시 "접근 권한 없음" 표시 ✅
5. 계정 B로 돌아와 계정 C 재활성화 → 계정 C 정상 접근 복구 ✅

---

## 시나리오 5: 기관 관리자 — 기관명 수정

1. 계정 B로 `/settings` 접속
2. 기관 정보 탭 → 기관명 변경 → 저장
3. 헤더 기관명 즉시 반영 ✅
4. 환자 초대 SMS 발송 → 변경된 기관명으로 발송 ✅

---

## 시나리오 6: 기관 관리자 — 직원 초대

1. 계정 B로 `/settings` 접속
2. 직원 초대 탭 → 이메일 + 역할 선택 → 초대 발송
3. 초대 이메일 도착 확인 ✅
4. 초대 수락 후 직원 목록에 추가 확인 ✅

---

## 시나리오 7: 슈퍼 어드민 — 전체 기관 조회

1. `mobys0416@gmail.com`으로 로그인
2. 헤더에 "관리자" 메뉴 표시 ✅
3. `/admin` 접속 → 전체 기관 목록 (기관명/직원수/생성일) ✅

---

## 시나리오 8: 슈퍼 어드민 — 기관 직원 권한 제어

1. `/admin` → 기관 X 선택 → 직원 목록
2. 특정 직원 비활성화 → 즉시 적용 ✅
3. 해당 직원 로그인 → 기관 X 접근 차단 ✅

---

## 시나리오 9: 엣지케이스 — 마지막 owner 비활성화 차단

1. 기관 Y의 유일한 owner인 계정 D로 로그인
2. `/settings` → 자기 자신 비활성화 시도
3. "마지막 관리자는 비활성화할 수 없습니다" 오류 ✅

2. "접근 권한이 없습니다" 또는 `/` 리다이렉트 ✅
