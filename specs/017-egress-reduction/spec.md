# Feature Specification: 이그레스(데이터 전송) 절감

**Feature Branch**: `017-egress-reduction` (작업: `claude/eo-sso-token-verify-w9v931`)
**Created**: 2026-06-29 · **Status**: Implemented
**Trigger**: Supabase 무료 이그레스 5GB 초과로 프로젝트 정지(2026-06-29). Pro 업그레이드로 복구 후 재발 방지.

## Summary

Supabase 이그레스(밖으로 나가는 데이터) 주범인 **이미지**를 줄인다. 세 축:

1. **업로드 압축**: 상담 이미지를 원본(수 MB) 그대로 저장하던 것을 **다운스케일(최대 1600px) + webp 인코딩**으로 압축 후 저장 → 저장 용량·조회 이그레스 동시 절감(~10배). 앞으로 들어오는 이미지에 즉시 효과.
2. **표시용 변환**: 목록/갤러리/본문의 `<img>`가 원본 대신 **작은 변환본**(Supabase render/image)을 받도록 URL 재작성 → 기존에 이미 올라간 큰 이미지에도 효과. + `loading="lazy"`로 오프스크린 로드 방지.
3. **안전장치**: 변환은 `NEXT_PUBLIC_IMG_TRANSFORM=off`로 즉시 끌 수 있음(이미지 변환 사용량이 문제되면 재배포 없이 원본 URL로 폴백; 압축·lazy는 유지).

## Requirements

- **FR-001** 상담 이미지 업로드(붙여넣기·드롭·파일·주석 전부 `uploadImage` 단일 경로) 전 압축(webp, 최대 1600px, q0.82). 이득 없으면 원본 유지, gif/비이미지 제외, EXIF 방향 반영.
- **FR-002** 본문 HTML 렌더 시 `<img>` src를 변환 URL로 재작성 + `loading="lazy"`(홈 피드·기록·상담이력·환자 포털).
- **FR-003** 이미지 갤러리/썸네일은 작은 변환본(width 300~400) + lazy(상담이력·환자 포털·상담 뷰).
- **FR-004** 변환은 env 플래그로 off 가능(폴백=원본). 압축·lazy는 항상 유지.
- **FR-005** 상담 데이터·화질(진료 사진 판독) 저하 없이 동작(1600px·q0.82는 판독 충분).

## Success Criteria

- **SC-001** 신규 업로드 이미지 저장·전송량이 원본 대비 수 배~10배 감소.
- **SC-002** 목록/갤러리에서 기존 이미지도 작은 변형본으로 전송(원본 미다운로드).
- **SC-003** 이그레스 재발 여유 확보(Pro 250GB 대비 충분한 헤드룸).

## Out of Scope

- 기존 원본 이미지의 일괄 재압축(백필) — 필요 시 후속 배치.
- 음성(spec 009) 이그레스 — 재청취는 온디맨드(클릭)라 수동적, 이번 범위 밖.
- CDN/캐시 헤더 세부 튜닝.

## Notes (운영)

- 표시용 변환은 Supabase "이미지 변환" 사용량을 씀(Pro 포함량 초과 시 과금). 소규모 의원 규모엔 이그레스 절감이 훨씬 큼. **모니터링**: Supabase → Reports에서 이미지 변환·이그레스 추이 확인. 문제 시 `NEXT_PUBLIC_IMG_TRANSFORM=off`.
