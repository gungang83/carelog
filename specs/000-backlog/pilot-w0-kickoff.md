## C-01 · 녹음 중 상담창 이탈/종료 시 파일 처리 ✅ 1차+2차 배포

- **출처**: 김도은 회의 / 메모
- **무엇이 불편했나**: "녹음 시작했다가 바로 상담창을 버리거나 끌 때 녹음 파일이 어떻게 처리되는지 확인 필요."
- **관찰/코드 대조**:

  - **"버리기" 버튼**(discard)이 확인 없이 즉시 삭제 → 오클릭 유실.
  - 빈 녹음(1KB 미만)은 자동 폐기 + 에러 안내(기존 진단 계측).
- **빈도/영향**: 가끔 · 영향: **기록 누락**(치명)
- **개선(1차 적용)**:
  - ✅ `beforeunload` 경고 — 녹음 중·처리 중이거나 저장 전 작성물/음성이 있으면 탭 닫기·새로고침 시 브라우저 확인창.
  - ✅ "버리기" 확인 다이얼로그 — 내용 있을 때만 `window.confirm`.
- **개선(2차 적용)**: 녹음 blob/작성물 **IndexedDB 임시영속화 + 재진입 복구**.
  - `lib/chair/draft-store.ts`: 단일 DRAFT 키, 본문·처방·참여자·체어·음성 blob 저장. 24h 만료. IndexedDB 미지원 시 조용히 무시.
  - 보드에서 1초 디바운스 자동저장(깨끗하면 미저장). 저장/버리기 시 정리.
  - 재진입 시 깨끗한 상태에서만 "이전에 작성하던 상담 기록이 있어요 · 복구하기/새로 시작" 배너. `RichTextEditor.setHTML`로 본문(HTML) 복원.
  - beforeunload는 "막는" 가드, IndexedDB는 "복구하는" 안전망 — 둘이 보완.

## C-07 · 참여자 선택 매끄러움 ✅ 배포

- **출처**: 김도은 회의 전사록 ("김도우 어디 있어")
- **진짜 원인(코드 대조)**: 단순 검색 불편이 아니라 **마스킹 충돌**. 피커가 후보·선택칩을 `maskName()`으로 가렸는데 규칙이 *3자↑ → 앞 2자+\**라, `김도은`·`김도우`가 **둘 다 `김도*`로 표시** → 구분·스캔 불가. 부차: 마스킹↔실명검색 불일치, 초성검색 없음, 26명 평면 칩더미(구획 라벨 없음).
- **개선(적용)**:
  - **피커 실명 노출** — 입력 단계는 직원 내부 도구·후보는 동료 의료진이므로 `maskName` 제거(동명 충돌 해소). 저장된 기록의 환자 노출 마스킹은 그대로 유지(분리돼 있던 걸 피커만 과하게 가렸던 것).
