### 이번 세션 처리(배포 대기)
- **[A2] 요약 맨 앞 환자 이름** — 녹취 첫머리에 이름이 있으면 요약 제목 다음 줄 "환자: <이름>". 미연결 카드 식별성↑. (basic/chunk/detailed/dental 프롬프트 반영)

- **[신규] 녹음 시작 안내문** — 녹음 중 배너: "녹음이 시작되었습니다 / 환자 성함 먼저·치아 부위(번호) 또렷이 말하면 정확히 기록". 회의의 '상담 규칙'(이름 먼저·치식 확실히)을 앱에 내장.

### 착수 예정 (다음 — 권장: spec 011로)
- **[A5+A6] 상담 기록 전체 열람·검색·필터 + 환자별 간략 보기** ★ "지금 착수"(대표 지시)
  - 홈 '기록·활동' 옆 **전체보기** → 리스트 + **정렬/필터/키워드 검색**(날짜·체어·미연결/연결·환자별).
  - **환자별 화면**: 연결 환자 열면 상담카드가 줄줄이 길게 나열돼 보기 힘듦(캡처) → **카드 간략 보기(열고닫기)** + 동일 필터/검색. 즉 환자별 화면 = 전체보기의 '환자 필터' 뷰.
  - 현재: 홈피드 `home-feed.tsx`는 10개+전체보기(단순 펼침, 필터 없음). 환자상세 `consultation-history.tsx`는 전체 펼침(접기 없음). → 둘에 정렬/필터/검색·접기 도입.

- **[A3] 연결완료 카드 옵션 노출** — ⚠️ 부분 완료. **환자 상세(`consultation-history.tsx`)엔 이미 전체복사·편집·SMS전송 있음**(세션 44). **남은 것 = 홈 피드의 '활동(연결완료)' 행**에도 인라인 전체복사/편집 노출(현재 로그 행만). 회의 맥락: "연결완료도 바로 떠야 덴트웹에 빨리 넣는다."
