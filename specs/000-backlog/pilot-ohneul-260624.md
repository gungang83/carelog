## O-1 · 다국어 실시간 통역·번역 ★ (신규 차별화 축)

- **출처**: 서범기 미팅 (08:56~14:37). 대표 "오늘 또 하나의 방향성이 결정됐어… 좋은 아이디어다."
- **문제**: 오늘의 치과는 외국인 신환이 유난히 많음. 원장이 영어로 신환 상담 → ① 통역사 없으면 20~30분 대기 ② 있어도 원장이 끝나고 2~3분 재설명 ③ 실장이 못 알아들으면 원장에게 재확인. 받아적기 에너지 큼. 지금은 서로 폰으로 번역 주고받느라 또 오래 걸림.
- **해법(방향)**:
  - 마이크 녹음 → 화면에 **원문 + 통역 동시 표시**. 양방향(영↔한, 일↔한 등). 원문 위·통역 아래, **버튼 토글**.
  - 동시통역까지 안 가도 **"녹음 + 사후 번역(텍스트화→번역)"만으로도 효용**(서범기: "동시통역 말고 녹음하는 거라도 되면").
- **근거 있는 자신감**: STT·번역 엔진 이미 성숙. 대표 "기술 발전을 내가 주도할 필요는 없고, 빠르게 발전하는 기술을 잘 적용하는 게 내 일". AI 엔진 기반이라 한글 고정 아님 → 다국어 자연 확장.
- **부수효과**: 외국인 상담 영상 = 외국인 마케팅 자산. 강남·지방 공히 외국인 급증.
- **난이도**: 중(엔진 적용·UI). **분류**: [x] **spec 승격 1순위 후보**
- **연결**: 기존 STT 파이프라인(상담보드) 확장. `specs/0XX-multilingual-interpret`(가칭).

## O-3 · 캡처 워크플로 (C-08 보강 — 차별점 명확화)

- **출처**: 서범기 미팅(파노라마 캡처) + 김도은 반박.
- **현황**: 캡처 도구 상시노출 → 영역선택 → 보드삽입(C-08). **단 김도은 반박**: "덴탈잡에서 문자로 파노라마 이미 보낼 수 있다" → 단순 이미지 전송은 차별점 아님.
- **차별점(재정의)**: **상담 본문 + 이미지(주석 포함) + 음성을 한 번에** 환자에게 전달 = 케어로그만의 강점. 캡처는 그 안의 한 부품.
- **외부 제약(기록)**: 덴트웹은 이미지 통째 붙여넣기 X. 이미지 복사 Ctrl+V 또는 덴트웹 사진보기 업로드로 우회.
- **분류**: [ ] C-08과 통합 — 차별점을 "통합 전달"로 잡아 spec 작성.

## 파일럿 운영 메모
- 오늘의 치과 = **2번째 파일럿**(예미안 병행). 외국인 多·말 많은 원장(신환검진)·직원 부족 → 효율화 여지 큼. 대표 "갖춰져 있지 않은 시스템인데 매출 높고 환자 많고 직원 모자라 → 효율화 여지 많다."

- 도입 톤: 원장에게 "컨설팅 업체(월 500~1000만) 대체" 가치 제안(서범기).
