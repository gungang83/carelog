## 왜 필요한가
대표가 "Carelog 프로필의 슈퍼어드민/설정 메뉴 구성을 EO 메뉴 구성을 참고해 맞추라"고 지시.
다온의 GitHub 접근 범위가 `gungang83/carelog` 한 곳으로 제한돼 EO 레포·UI를 직접 참고할 수 없어,
EO를 운영하는 휴고에게 메뉴 구조를 요청한다(추측 모방 = 오정렬 위험 회피).

## 받고 싶은 것 (구체)
1. **프로필/계정 드롭다운** — 항목 순서와 라벨(예: 내 계정 · 설정 · 관리자 · 로그아웃 …).
2. **설정 메뉴의 범위와 섹션** — 워크스페이스 단위인지, 개인 단위인지, 하위 섹션 목록.
3. **슈퍼어드민(최고관리자) 메뉴** — 노출 조건(역할/계정), 하위 기능(기관 관리·요금·기능 게이트 등).
4. **설정 ↔ 슈퍼어드민 경계 원칙** — "이건 설정, 저건 슈퍼어드민" 판단 기준.

## 현재 Carelog 상태 (참고)
- `/settings` (설정) = 현재 워크스페이스: 요금제 · 기관 프로필 · 체어 관리 · 멤버/직원 관리·초대 · 내 상담 기록 · 알림.
- `/admin` (슈퍼어드민 = 최고 관리자 패널) = 전체 기관·직원 총괄 + **워크스페이스별 실험실(Engine Lab) 토글**. `isSuperAdmin` 계정만.
- 이번 변경: 프로필 드롭다운 최상단에 **슈퍼어드민** 진입점 추가(super 계정만 노출).
