## §A. EO 메뉴 구조 (휴고 작성 원문 보존)

### ① 프로필 드롭다운 (EO `UserMenu.tsx`) — 위→아래
1. 현재 계정 정보(이름·이메일) — 항상. 임퍼소네이트 중이면 `원본: {email}` 표시
2. 계정 전환 — `canSwitch`(전환 권한자). 리스트+뱃지(본계정/데모/슈퍼어드민/원장/매니저/직원) → `POST /api/superadmin/switch-account`
3. 슈퍼어드민 — `isSuperAdmin && !isImpersonating` → `/superadmin`
4. 설정 — 항상 → `/settings`
5. 고객센터 — 항상 → `/support`
6. 로그아웃 — 항상(구분선 위)
- 설계: 순서=빈도·권한 역순(위=권한자만, 아래=전원). 슈퍼어드민은 전환 중 숨김(죽은 링크 제거).

### ② 설정 (`/settings`) — 탭 + 접이식, 권한 노출
`notification`알림(전원) · `clinic`근무·진료(관리자) · `jobs`직군(원장) · `permissions`권한관리(관리자) · `announcements`공지(관리자) · `lab`🧪실험실(전원). `visibleTabs = tabs.filter(t => !t.adminOnly || canManagePermissions)`
- 설계: 설정=내 워크스페이스 운영설정만(타 고객사·전역 없음). staff는 알림·실험실 2탭만.

### ③ 슈퍼어드민 (`/superadmin`) — EO 운영사 전용(고객사엔 비노출)
접근(`eo-operator.ts`): `isEoOperator`. owner/operator 2단. 메뉴: 개요(전원)·워크스페이스(owner)·고객문의(operator)·공지(owner)·핸드오프큐(owner)·운영진설정(owner)·협회/의무항목(owner). 개요=고객사수·미처리문의·SLA위반·가입추이·플랜분포. 부트스트랩 이메일=DB없이 항상 owner.

### ④ 설정 ↔ 슈퍼어드민 경계
한 줄: **"내 고객사 운영설정"=설정 / "EO 플랫폼·전체 고객사 운영"=슈퍼어드민.**
판단규칙: ①한 고객사 안에서만 의미→설정 ②여러 고객사 걸침/EO가 고객사 관리→슈퍼어드민 ③권한분리(고객사 직원등급→설정 / EO 운영진→슈퍼어드민) ④주체로 가름.
- Carelog 권장: 3-tier 권한 메뉴 필터 · 드롭다운 2분할(권한자 위/전원 아래) · 운영자콘솔 완전분리+진입가드+임퍼소네이트중 숨김 · 경계=데이터범위(단일 vs 전체).

## §B. Carelog 매핑 · 조치 (다온)

**핵심: EO 4섹션이 Carelog 현재 구조를 대체로 검증.** 이미 정렬된 것 + 적용할 것:

| EO 원칙 | Carelog 현재 | 판정 |
|---|---|---|
| 드롭다운 2분할(권한자 위/전원 아래) | 슈퍼어드민(super, 상단 구분선) → 내 상담기록·설정·서비스소개 → 로그아웃 | ✅ 정렬됨 |
| 슈퍼어드민 = 플랫폼/전체 고객사 | `/admin`(전 기관·직원·실험실 토글), `isSuperAdmin` 게이트 | ✅ 정렬됨 |
| 설정 = 내 워크스페이스만 | `/settings`(요금제·기관·체어·멤버·알림), `isOwnerOrAdmin` 게이팅 | ✅ 정렬됨 |
| 3-tier 권한 | staff / owner·admin / isSuperAdmin | ✅ 대응됨 |
| 경계=데이터범위(단일 vs 전체) | 설정=단일기관 / 슈퍼어드민=전체 | ✅ 동일 원칙 |
| 임퍼소네이트 중 슈퍼어드민 숨김 | Carelog는 임퍼소네이트 기능 없음 | N/A |
| 계정 전환 섹션 | Carelog는 멀티 계정전환 없음(기관 스위처는 별도) | N/A |

**이번 적용(저위험):**
- 슈퍼어드민 페이지 H1 "최고 관리자 패널" → **"슈퍼어드민"**(메뉴·EO 라벨 일치).
- 경계 원칙(④)을 `docs/architecture.md`에 명문화 → 향후 메뉴 추가 시 기준.

**선택 후속(미적용, 대표 결정 대기):**
1. 설정을 **탭 구조**로(EO ②) — staff는 알림만, 관리자는 전체. (중규모 리팩터)
2. `/admin` → `/superadmin` 라우트 리네임(현재 라벨만 슈퍼어드민, 경로는 /admin). (링크 영향)
3. 슈퍼어드민 **개요 대시보드** 확장(현재 카운트 3종 → 문의·플랜분포 등). EO는 EO 운영사라 SLA/문의가 핵심이나 Carelog는 단일 운영(다온)이라 우선순위 낮음.
4. 임퍼소네이트/계정전환 — Carelog 현재 불필요(N/A).
