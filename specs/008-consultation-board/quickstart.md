# Quickstart: 상담보드 검증 절차

수용 시나리오(spec) ↔ 수동 검증 매핑. 환경: Vercel 프로덕션(또는 로컬), PC/안드로이드 Chromium, 로그인 직원.

## 사전 준비
- 멤버가 여러 명(가능하면 20명+) 등록된 기관(예: 예미안)
- 체어 몇 개 등록
- 마이크 사용 가능 기기

## P1 — 일단 녹음부터 (record-first)
1. 홈에서 체어·참여자 **아무것도 안 고른 채** "바로 녹음/상담 기록 시작" 1탭
   → ✅ 즉시 녹음 시작, 타이머 흐름 (SC-001·002, AS1)
2. 보드를 닫았다가 다시 열기
   → ✅ 녹음 계속 진행 중, 경과 시간 유지 (FR-003, AS4)
3. [중지] 누르기
   → ✅ 전사 결과가 본문에 채워지고 편집 가능 (AS2)
4. 체어 하나 고르고 [저장]
   → ✅ 그 체어에 귀속 저장 + "저장됨 ✓" (FR-005, AS3)
5. 미연결 기록 목록/실시간 알림에 정상 반영
   → ✅ 기존 동작 회귀 없음

## P2 — 참여자 부담 없이 붙이기
1. 보드 진입 시 참여자 영역 확인
   → ✅ 로그인 본인('나') 기본 포함, 최근 함께한 사람 상단 (AS1)
2. 참여자 검색창에 이름 일부 입력
   → ✅ 목록 즉시 좁혀짐 (AS2, SC-003)
3. 비진료·미분류(예: 대표·테스트) 항목 확인
   → ✅ 기본 노출 후순위/접힘, 단 검색으로는 도달 가능 (AS3)
4. 참여자 하나도 안 고르고 저장
   → ✅ 정상 저장 (FR-010, AS4)

## P3 — 한 화면에서 상담 진행 (종합 보드)
1. 녹음 진행 중 본문 타이핑 + 이미지에 그림 주석 추가
   → ✅ 녹음 끊김 없이 입력 보존 (SC-005, AS1)
2. 처방 선택 + 체어 선택까지 한 화면에서 수행 후 저장
   → ✅ 화면 전환 0, 한 보드에서 저장 완료 (SC-006, AS2)
3. 저장된 처방이 기록에 반영
   → ✅ (AS3)

## 엣지 검증
- 녹음 중 보드 닫기→복귀: 녹음 유지
- 저장 전 보드 닫기: 진행 중 draft 보존(재진입 시 이어서) (FR-016)
- 빈 녹음(가능 시 iOS): "녹음 N초·KB" 진단 안내, 강제 저장 안 함
- 저장 시 체어 미선택: 빠른 체어 선택 유도(차단 아님)

## 빌드/배포 게이트
- `npm run build` 그린(TypeScript 통과)
- DB 변경 없음 → 마이그레이션 적용 불요(MVP)
- 신규 사용자(1년차) 안내 없이 녹음→저장 1회 성공 (SC-007) — 파일럿에서 확인
