# Feature Specification: 상담보드 (Consultation Board) — record-first 통합 상담 기록

**Feature Branch**: `008-consultation-board`
**Created**: 2026-06-18
**Status**: Draft
**Input**: User description: "상담보드 — 급하게 녹음 먼저 시작하고, 녹음이 도는 동안 체어·참여자·텍스트·그림을 병행해 채우는 통합 상담 기록 캔버스"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 일단 녹음부터 (record-first) (Priority: P1)

진료가 급하게 시작된다. 직원은 체어나 참여자를 고를 틈 없이 **단 한 번의 동작으로 녹음을 즉시 시작**한다. 상담이 끝나면 녹음을 중지하고, 그제서야 어느 체어였는지 정하고 저장한다. 체어·참여자·내용 정리는 모두 *녹음 시작 이후*의 일이다.

**Why this priority**: 현재는 "참여자·체어 선택 → 오버레이 → 녹음 시작"의 순차 게이트라, 급한 진료에서 선택을 강요받다 기록 자체를 놓친다. 이 슬라이스만으로도 "급할 때 일단 남긴다"는 핵심 가치가 완성되며, 인력 부족 현장의 기록 누락을 직접 줄인다.

**Independent Test**: 체어·참여자를 전혀 고르지 않은 상태에서 녹음을 시작→상담→중지→저장까지 끝낼 수 있고, 저장 시점에만 체어를 정하면 기록이 보존됨을 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 직원이 홈 화면에 있고 아무 체어도 고르지 않은 상태에서, **When** 녹음 시작을 한 번 누르면, **Then** 즉시 녹음이 시작되고 타이머가 흐른다(선택 단계 없음).
2. **Given** 녹음이 진행 중이고 아직 체어가 미확정인 상태에서, **When** 녹음을 중지하면, **Then** 전사 결과가 본문에 채워지고 편집 가능한 상태가 된다.
3. **Given** 전사된 본문이 있고 체어가 미확정인 상태에서, **When** 체어 하나를 고르고 저장하면, **Then** 상담이 그 체어에 귀속되어 저장되고 성공 메시지가 표시된다.
4. **Given** 녹음 중 화면(보드)을 닫거나 다른 화면으로 이동해도, **When** 다시 돌아오면, **Then** 녹음은 끊기지 않고 계속 진행 중이며 경과 시간이 유지된다.

### User Story 2 - 참여자 부담 없이 붙이기 (Priority: P2)

직원이 상담 참여자를 지정하려는데 기관 멤버가 수십 명이라 스크롤로 찾기 어렵다. 보드는 **로그인한 '나'를 기본으로 깔아두고**, 이름 검색으로 좁히며, **최근 함께 기록한 사람을 위에** 보여준다. 상담과 무관한 항목(대표·미분류·테스트성)은 기본 목록에서 빠져 있다. 참여자 지정은 *선택*이고, 안 골라도 저장된다.

**Why this priority**: 실제 현장(예미안)에서 참여자가 26명이라 선택이 가장 큰 마찰점이다. 이 슬라이스는 P1과 독립적으로도 기존 흐름에 즉시 가치를 준다(빠른 참여자 지정).

**Independent Test**: 멤버가 20명 이상인 기관에서, 검색·'나' 자동·최근 노출만으로 참여자 1명을 3초 안에 지정하거나, 아예 지정하지 않고도 저장이 완료됨을 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 멤버가 20명 이상인 기관에서 보드를 열면, **When** 참여자 영역을 보면, **Then** 로그인한 본인이 기본 참여자로 들어가 있고 최근 함께한 사람이 상단에 보인다.
2. **Given** 참여자 영역에서, **When** 이름 일부를 입력하면, **Then** 목록이 해당 이름으로 즉시 좁혀진다.
3. **Given** 비진료·미분류·테스트성 역할의 항목이 있을 때, **When** 기본 참여자 목록을 보면, **Then** 그 항목들은 기본 노출에서 제외(또는 후순위)되어 있다.
4. **Given** 참여자를 한 명도 지정하지 않은 상태에서, **When** 저장하면, **Then** 저장이 정상 완료된다.

### User Story 3 - 한 화면에서 상담 진행 (종합 보드) (Priority: P3)

직원은 녹음이 도는 동안 같은 화면에서 본문을 타이핑하고, 엑스레이/사진에 그림으로 표시하고, 처방을 고르고, 체어를 정한다. 화면 전환 없이 **하나의 보드에서 상담을 진행하고 한 번에 저장**한다.

**Why this priority**: P1·P2가 마찰을 없앤 위에 얹는 "통합 캔버스"로, 상위 비전의 체어사이드 상담창(Carelog Terminal)을 구현한다. 단독으로도 "녹음+메모+그림을 한 화면에서"라는 가치를 준다.

**Independent Test**: 녹음이 진행되는 동안 본문 입력·이미지 주석·처방 선택을 모두 수행해도 녹음이 끊기지 않고, 한 화면에서 저장까지 완결됨을 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 녹음이 진행 중인 보드에서, **When** 본문에 텍스트를 입력하거나 이미지에 그림 주석을 추가하면, **Then** 녹음은 중단 없이 계속되고 입력 내용이 보존된다.
2. **Given** 녹음·본문·그림·처방·체어가 채워진 보드에서, **When** 저장을 누르면, **Then** 한 화면 안에서 저장이 완료되고 이어서 환자 연결로 진행할 수 있다.
3. **Given** 보드에서 처방을 선택하면, **When** 저장하면, **Then** 처방이 상담 기록과 함께 저장된다.

---

### Edge Cases

- **녹음 중 이탈/복귀**: 보드를 닫거나 다른 화면으로 가도 녹음은 중지 전까지 지속되고, 복귀 시 진행 상태·경과 시간이 유지된다.
- **저장 전 이탈**: 아직 저장하지 않은 진행 중 상담(임시 세션)은 사용자가 명시적으로 버리기 전까지 보존되어, 실수로 닫아도 내용이 사라지지 않는다.
- **빈 녹음(예: iOS 화면잠금)**: 녹음이 비어 있으면 기존 진단 안내("녹음 N초·KB")로 원인을 알리고 기록을 강제 저장하지 않는다. (녹음 신뢰성 자체는 별건으로 파킹)
- **여러 체어 동시 진행**: 한 직원이 한 번에 하나의 상담 세션을 진행하는 것을 기본으로 한다. 동시 다중 녹음은 본 기능의 범위 밖(현 멀티체어 동작은 회귀시키지 않음).
- **저장 시 체어 미선택**: 저장하려면 체어 하나가 필요하다 — 미선택 시 빠른 체어 선택을 유도한다(저장 차단이 아니라 1탭 선택).
- **참여자 데이터 노이즈**: 비진료·미분류·테스트성 멤버가 섞여 있어도 기본 참여자 후보에서 제외/후순위화한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 사용자는 체어·참여자 선택 없이 **한 번의 동작으로 녹음을 즉시 시작**할 수 있어야 한다.
- **FR-002**: 녹음이 진행되는 동안 시스템은 사용자가 **본문 입력·이미지 그림 주석·체어 선택·참여자 지정·처방 선택을 병행**하도록 허용해야 하며, 이 작업들이 녹음을 중단시키지 않아야 한다.
- **FR-003**: 녹음은 사용자가 보드/화면을 닫거나 다른 작업으로 이동해도 **중지 전까지 지속**되어야 하며, 복귀 시 진행 상태·경과 시간이 유지되어야 한다.
- **FR-004**: 사용자는 녹음을 중지할 수 있고, 중지 시 **전사 결과가 본문에 합쳐져 편집 가능**해야 한다.
- **FR-005**: 시스템은 **저장 시점에 상담을 체어에 귀속**시켜야 하며, 저장 전까지는 체어가 미확정인 상태를 허용해야 한다.
- **FR-006**: 참여자 지정 시 시스템은 **로그인한 사용자('나')를 기본 참여자로 우선 포함/노출**해야 한다.
- **FR-007**: 참여자 목록은 **이름 검색으로 즉시 좁혀**져야 한다.
- **FR-008**: 시스템은 **최근 함께 기록한 참여자를 상단에 우선 노출**해야 한다.
- **FR-009**: 시스템은 상담 참여자로 부적합한 항목(비진료·미분류·테스트성 역할)을 **기본 노출에서 제외 또는 후순위화**해야 한다.
- **FR-010**: 참여자 지정은 **선택사항**이며, 미지정 상태로도 저장 가능해야 한다.
- **FR-011**: 모든 저장(mutation)은 서버에서 처리되어야 하고, 저장 후 관련 목록이 즉시 최신화되어야 한다. (헌법 II)
- **FR-012**: 보드는 진료 본문·환자정보를 다루되, **실시간 알림 신호에는 환자정보·진료본문이 실리지 않아야** 한다(체어·도착 사실만). (헌법 I)
- **FR-013**: 저장된 상담은 기존처럼 **환자 연결**로 이어질 수 있어야 한다.
- **FR-014**: 상담 작성의 모든 단계는 **하나의 보드 화면 안에서 완결**되어, 별도 화면 전환을 요구하지 않아야 한다.
- **FR-015**: 저장 결과는 **명시적 성공/실패 메시지(한국어)**로 사용자에게 표시되어야 한다. (헌법 III)
- **FR-016**: 아직 저장하지 않은 진행 중 상담은 사용자가 명시적으로 버리기 전까지 **보존**되어야 한다(실수 닫힘 방지).

### Key Entities *(include if feature involves data)*

- **상담 세션(작업 단위)**: 저장 전 진행 중인 한 건의 상담. 녹음 상태, 전사 본문, 임시 작성 텍스트/그림, 선택된 체어(미확정 가능), 참여자, 처방을 담는다. 저장되면 기존 상담 기록(체어 귀속, 환자 미연결)이 된다.
- **참여자 후보**: 기관 멤버 디렉터리. 이름·역할을 가지며, 검색·필터·최근순 정렬의 대상. 상담 적합/부적합(비진료·테스트성) 구분이 필요.
- **체어**: 상담이 저장 시 귀속되는 위치 라벨. 저장 전까지 미확정 가능.
- **전사 결과**: 녹음 중지 후 생성되어 본문에 합쳐지는 텍스트.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 녹음 시작까지 필요한 사용자 동작이 **1회(탭)**로 줄어든다(현재 3단계: 참여자→체어→녹음).
- **SC-002**: 직원이 마음먹은 뒤 **5초 이내**에 녹음이 시작된다(선택 단계가 가로막지 않음).
- **SC-003**: 멤버 20명 이상 기관에서 참여자 1명 지정이 **검색/자동포함으로 3초 이내** 가능하거나, 미지정으로도 저장이 완료된다.
- **SC-004**: 파일럿 기간 기준, "상담했으나 기록하지 못함"(기록 누락) 사례가 **측정 가능하게 감소**한다.
- **SC-005**: 녹음 도중 본문 입력·그림 주석·처방 선택을 수행해도 **녹음 끊김 0건**.
- **SC-006**: 한 건의 상담을 기록·정리·저장하는 동안 **화면 전환 0회**(단일 보드에서 완결).
- **SC-007**: 신규 사용자(예: 1년차 직원)가 안내 없이 **첫 시도에 녹음→저장**을 성공한다.

## Assumptions

- **체어는 저장 시 필수**: 녹음 시작은 체어 없이 가능하되, 저장 시 체어 1개에 귀속한다(빠른 1탭 선택). 이는 기존 데이터모델(상담=체어 귀속, 환자 미연결)과 스키마를 변경 없이 유지하기 위함이다. "체어 없이도 저장(미배정)"은 본 스펙 범위 밖으로 둔다.
- **단일 작성자 귀속**: 작성자는 로그인한 직원으로 자동 귀속한다(기존 동작 재사용). 공용계정 다인 구분은 범위 밖.
- **기존 자산 재사용**: 백그라운드 녹음(중지 전까지 지속), 리치 텍스트 입력, 이미지 그림 주석, 처방 선택, 참여자 디렉터리, 작성자 귀속은 이미 존재하는 기능을 재사용한다(헌법 IV — 새 추상화 최소화).
- **전사 합류 시점**: 전사 결과는 녹음 중지 시 본문에 합쳐진다(실시간 받아쓰기 누적은 범위 밖, 기존 동작 유지).
- **대상 플랫폼**: PC/안드로이드 Chromium 우선. iOS의 백그라운드 녹음 신뢰성은 별건(파킹)으로, 본 스펙은 그 신뢰성을 해결 대상으로 삼지 않는다.
- **참여자 적합성 기준**: "상담 참여자 후보"는 진료/현장 역할 중심이며, 비진료(예: 대표)·미분류·테스트성 항목은 기본 노출에서 제외/후순위화한다(정확한 역할 매핑은 plan 단계에서 확정).

## Out of Scope

- 환자 자동 매칭/연결 추천 (별도 기능)
- 공용계정 다인 작성자 구분 (이후 EO SSO와 함께)
- iOS 백그라운드 녹음 신뢰성(빈 녹음) 해결 (파킹된 별건)
- 실시간 받아쓰기(녹음 중 본문 실시간 갱신)
- 체어 미배정 저장(미배정 보관함)
