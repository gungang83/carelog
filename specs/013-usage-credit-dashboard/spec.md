# Feature Specification: 사용량 · 크레딧 대시보드 (슈퍼어드민)

**Feature Branch**: `013-usage-credit-dashboard` (작업: `claude/eo-sso-token-verify-w9v931`)
**Created**: 2026-06-28 · **Status**: Implemented
**Benchmark**: EO `gungang83/EO@main` — `/superadmin/menu-usage`(spec-075) + 크레딧(spec-011) 읽기전용 교차검증

## Summary

슈퍼어드민이 **(A) 메뉴(화면) 사용량**과 **(B) AI 크레딧 사용량**을 한 화면에서 확인한다.
EO의 메뉴 사용량 분석 도구를 케어로그 멀티테넌트(`institution_id`)로 포팅하고, 여기에
**누가 / 얼마나 / 어떤 기능에서 크레딧을 썼는지** 상세 + 통계를 더한다.

- **메뉴 사용량**: 화면 진입을 일별 집계(클릭당 row 폭증 방지) → 기관별·메뉴별(역할분해)·미사용 메뉴.
- **크레딧 사용량**: AI 전사 기능 호출마다 기능별 단가를 차감 로그로 적재 → 기능별·사용자별·기관별 사용 + 잔액 + 최근 내역.
- **크레딧 모델**: 시뮬레이션(슈퍼어드민 부여 + 차감 로그). **잔액 부족해도 전사를 차단하지 않는다**(임상 안정성 우선).

## User Scenarios

### US1 — 메뉴 사용량 확인 (슈퍼어드민)
슈퍼어드민이 `/admin/usage` 메뉴 탭에서 기간(7/30/90/365일)·기관 필터로 화면별 진입 수를 본다.
어떤 기관/역할이 어떤 화면을 얼마나 쓰는지, 아무도 안 쓰는 메뉴는 무엇인지 파악한다.

### US2 — 크레딧 사용량 확인 (슈퍼어드민)
크레딧 탭에서 총 사용 크레딧, **기능별**(빠른 메모·상세 요약·용어 보정·다국어·청크 등), **사용자별 상위**,
**기관별** 사용량, 기관별 **잔액**, **최근 사용 내역(일시·기관·사용자·기능·변동·잔액)**을 확인한다.

### US3 — 크레딧 충전 (시뮬레이션)
슈퍼어드민이 기관을 선택해 크레딧을 부여(또는 음수로 조정)한다. 부여는 원장에 'grant'로 기록된다.

## Requirements

- **FR-001** 화면 진입 시 `menu_id`를 비동기 수집(sendBeacon). 세션·기관·역할은 서버가 신뢰원으로 확인. 화이트리스트 메뉴만 기록.
- **FR-002** 수집은 `(institution, user, menu, day)` 단위 카운트 증가(원자적). KST 기준 일자.
- **FR-003** AI 전사 기능 호출 성공 시 기능별 단가를 `credit_log`에 차감 기록. created_by=사용자 이메일, feature=기능 키.
- **FR-004** 크레딧 차감은 **비차단·비실패**: 잔액과 무관하게 흐름을 막지 않으며(잔액 음수 허용) 기록 실패가 전사를 막지 않는다.
- **FR-005** 메뉴 요약 API: 총합·활성사용자·기관별·메뉴별(역할분해)·미사용. 슈퍼어드민 전용.
- **FR-006** 크레딧 요약 API: 총 사용/충전·기능별·사용자별 상위·기관별·잔액·최근 내역. 슈퍼어드민 전용.
- **FR-007** 충전 API: 슈퍼어드민이 기관별 크레딧 부여. 원장에 grant 기록.
- **FR-008** 모든 집계/충전은 서버 권위(Route Handler + service_role). 클라이언트는 표시·필터만.
- **FR-009** 사용량 테이블은 직원 PII(이메일) 외 환자 PII를 담지 않는다. 슈퍼어드민만 열람.

## Success Criteria

- **SC-001** 슈퍼어드민이 단일 화면에서 메뉴/크레딧을 기간·기관별로 즉시(<1s) 확인.
- **SC-002** 크레딧 사용을 "누가·얼마나·어떤 기능"으로 분해 가능.
- **SC-003** 메뉴 추적·크레딧 기록이 일반 사용자 흐름(전사·네비게이션)에 회귀를 일으키지 않음.

## Out of Scope

- 실결제·요금제 연동(후속 빌링 spec). 지금은 시뮬레이션.
- 잔액 기반 기능 차단(임상 안정성 결정으로 제외).
- 네비게이션 외 세밀 이벤트(버튼 클릭 등) 추적.
- 토큰/원가 단위 실측(크레딧은 기능별 고정 단가).
