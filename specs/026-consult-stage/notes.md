# spec 026 (기획 제안) — 상담 스테이지: 자료를 "보여주고 · 그리고 · 기록에 담는" 한 동선

> 상태: **기획 제안 — 대표님·김도은·송정현 검토용.** 2026-07-08 회의(김도은 안건) 후속.
> 작성: 다온 · 2026-07-08 · 선행: spec 025(자료 라이브러리+픽커+캔버스, 배포됨)

## 0. 문제를 한 문장으로

> "상담용 이미지·동영상을 **어디서 열고**, 그 위에 **마우스나 펜으로 그려가며** 상담하고, 그걸 **차트에 정리해 넣고 환자에게 전달**하는 동선을 어떻게 만들까?" (송정현)

spec 025는 자료를 '기록(에디터)에 삽입'하는 것까지 만들었다. 이번 제안은 그 앞단 —
**환자와 함께 보는 순간**을 설계한다.

## 1. 핵심 제안 — "스테이지" 레이어

보여주기용 화면과 기록용 화면을 별도 제품으로 나누지 않고, **버튼 하나로 오가는 두 얼굴**로 만든다.

```
[상담 진행: 녹음 중 · 에디터]
      │  📚 자료에서 선택 → "크게 열기"
      ▼
[스테이지] = 전체화면 자료 뷰어 + 그리기
      · 이미지: 위에 펜/마우스로 그리며 설명 (기존 ImageAnnotator 재사용 —
        펜·화살표·도형·텍스트·지우개·핀치줌·터치 이미 지원 ✅)
      · 동영상: 재생 (v1은 링크 자산)
      │
      ├─ [기록에 담기] → 그린 상태 그대로 스냅샷이 에디터에 삽입 (설명의 증거가 기록이 됨)
      └─ [그냥 닫기]  → 상담 계속
      ▼
[상담 기록] = 담긴 스냅샷 + 녹음 요약 + 메모  →  차트 복사 · 환자 포털/SMS 전달
```

**왜 이 구조인가**: "설명하며 그린 그림"이 곧 "환자 상담 레포트"가 된다. 보여주기→기록→전달이
별도 작업이 아니라 한 흐름의 부산물. 송정현 고민의 세 요구(보여주기 편의·차트 정리·환자 전달)를 한 동선이 해결.

## 2. 3가지 상담 환경별 대응 (송정현 프레임)

| 환경 | 입력 | 스테이지 대응 |
|---|---|---|
| ① 체어 + 정면 모니터 | 마우스만 | 전체화면 + **큰 버튼**(키보드 불필요). 마우스 드로잉. 픽커→크게 열기→담기 전부 클릭만으로 |
| ② 체어 + 태블릿 | 펜/터치 | 같은 스테이지 — annotator가 터치·핀치줌 기지원. 펜 드로잉 자연스러움 |
| ③ 상담실 + 컴퓨터 | 키보드 | 에디터 중심(spec 025 캔버스) + 필요할 때만 스테이지. 텍스트 정리 병행 |

→ **기기별 화면을 따로 만들지 않는다.** 하나의 스테이지가 입력 수단만 다르게 받는다(범용성).

## 3. 동영상 — "트는 것"이 아니라 "보내는 것" (김도은 우려 반영)

- 체어타임 악화 방지 원칙: **영상의 기본 활용 = 전송**. 상담 기록에 영상 링크 카드를 삽입 →
  환자 포털/SMS로 전달 → 대기실·귀가 후 시청. 스테이지 재생은 가능하되 보조.
- v1 자산 형태: **외부 링크(YouTube 한정공개 등)** — 파일 업로드는 스토리지·이그레스 부담 커서 후속.
  `consult_assets`에 `kind`('image'|'video_link') + `link_url` 확장.
- 주제별 시리즈(치경부 파절/마모, 충전/인레이/크라운/라미네이트)는 카테고리 체계 그대로 수용
  (`lib/consult-assets.ts` config에 세분 카테고리 추가만).

## 4. 동의서 · 치료비/견적

- **동의서**: 스테이지에서 열어 설명 + **펜 서명** → 서명본 스냅샷이 상담 기록에 첨부(환자 포털에서 사본 열람).
  ⚠️ 전자 동의의 **법적 효력·설명의무 입증 요건은 별도 검토 필요**(의료법·전자서명법 — 자문 권장).
  v1은 "서명 이미지가 붙은 상담 기록"까지, 법정 동의서 대체 주장은 검토 후.
- **치료비/견적**: v1은 견적 화면 캡처·표 이미지를 자산으로(당장 가능). 구조화 견적 블록(항목·금액·합계
  입력→기록 삽입→환자 전달)은 별도 스펙 후보(EO 회계와의 관계도 정리 필요).

## 7. 열린 질문 (검토 요청)

1. 스테이지 진입점: 픽커 안 "크게 열기"만으로 충분한가, 기록에 이미 삽입된 이미지도 클릭→스테이지로 열 수 있어야 하나? (다온: **둘 다** — 특히 후자가 재상담 때 유용)
2. 스냅샷 정책: "기록에 담기" 시 원본과 그린 본을 각각 남길 필요가 있나? (다온: 그린 본만 — 원본은 라이브러리에 있음)
3. 동영상 초기 소스: 김도은 팀이 만들 자체 영상 vs 외부 콘텐츠? (저작권 — 외부는 링크만)
4. P1 착수 시점: 예미안이 spec 025(라이브러리)를 먼저 1~2주 써보고 P1을 조정할지, 바로 이어 붙일지.
