# 예미안치과 내부 파일럿 — Carelog 도그푸딩 (4주)

> **상태**: 설계안 (2026-06-18, 다온 PM) · **대상 기관**: 예미안치과의원(`0e4e85d6`)
> **성격**: 내부 사용자 테스트 + 온보딩 + 주간 개선 루프. 외부 베타 이전 단계.

## 1. 배경 — 왜 지금, 왜 이 사람

예미안은 인력이 빠듯해 **상담은 이뤄지지만 그 내용을 충분히 기록으로 남기지 못하는** 경우가 잦다.
이 "기록의 빈칸"이 Carelog(STT → AI 요약 → 환자 전달)가 정확히 메우려는 통증이다.

대상자 **김도은(1년차 치위생사)**은 상담을 직접 주도할 일은 적지만 **상담 내용을 기록할 일은 많다.**
1년차 = *마찰 최대* 사용자라, 신입이 Carelog로 기록을 빠짐없이 해내면 그 자체가 가장 강한 제품 검증이다.

### 북극성 (이 파일럿이 증명하려는 것)
> "인력이 부족한 현장에서도, 신입 치위생사가 상담 내용을 **빠짐없이·부담 없이** 기록으로 남긴다."

## 2. 대상자 · 기간 · 보상

| 항목 | 값 |
|---|---|
| 대상자 | 김도은 (예미안 1년차 치위생사, 주 기록자) |
| 기간 | 4주 (0주 킥오프 + 1~4주 주간 1시간 회의) |
| 보상 | **수완홀딩스 지급, 세전 12만 원 / 4주 전체** (주 3만 상당) |
| 이후 | 효과·부담 평가 후 연장 여부 결정 (열어둠) |

> 보상은 **회의 + 솔직한 피드백**에 대한 사례. 일상 사용·추가 과업에 대한 대가가 아니다(§6 과부하 가드레일).
> 회계/세무 처리(근로소득 vs 기타소득)는 수완홀딩스 측에서 확인.

## 3. 성공 기준 (정성 + 정량 병행)

한 명의 목소리만 들으면 편향된다. 김도은의 체감(정성)과 사용 데이터(정량)를 같이 본다.

**정성 (김도은 체감)**
- "상담 기록에 드는 시간·스트레스가 줄었나"
- "다른 기기로 옮겨가며 쓰는 게 번거롭지 않았나"
- "못 쓰겠다고 느낀 순간"이 어디였나 (← 가장 중요한 데이터)

**정량 (우리가 관찰·계측으로 짊어진다)**
- 일별 녹음/기록 건수 ↑
- 기록 누락(상담했지만 미기록) ↓ — 김도은 자기 보고 + 활동 로그 대조
- 녹음 실패율 (이미 배포한 진단 계측 활용)
- 핵심 동작 탭 수 (원탭 녹음 정착 여부)

## 4. 워크스트림 A — 멀티 기기 세팅 (0주, 가장 먼저)

김도은이 자기 폰만이 아니라 **원내 노트북·태블릿·체어 PC**에서 두루 쓰도록 세팅한다.
이 세팅이 곧 **원탭 녹음**과 **spec 007 실시간 알림**을 동시에 살린다.

### 기기별 온보딩 체크리스트 (각 기기 1회)
- [ ] 브라우저는 Chromium 계열 (체어 PC/노트북 = Chrome/Edge 권장)
- [ ] 김도은 **개인 계정으로 로그인 유지** (작성자 귀속이 항상 김도은 → 단순·정확)
- [ ] **PWA 설치**(홈 화면 추가) — 마이크 권한을 1회만 허용하면 이후 재요청 없음
- [ ] 마이크 권한 **허용**(테스트 녹음 1회)
- [ ] **알림 소리 켜기**(🔔 1회 활성화) — 다화면 실시간 알림용
- [ ] (선택) Web Push 구독 허용
- [ ] **기기 = 체어 매핑**: 그 기기에서 해당 체어로 **한 번 기록** → 이후 그 기기는 그 체어를 기억(`localStorage`),
      홈에서 **"{체어명} 바로 녹음" 1탭** 진입
- [ ] **화면 자동잠금(예: 5분)** 설정 — 공용 화면에 환자 정보 노출 방지(헌법 I)

### 기기 선정 가이드
- **핵심 기록 기기 = 체어 PC(크로미움)** — 소리·푸시·백그라운드 제약 없음, 원탭 1:1 매핑 최적
- **아이패드(iOS)는 보조** — 소리·푸시·백그라운드 녹음 제약(spec007 v1 iOS 범위 밖).
  단, 아이패드를 쓰면 iOS 빈 녹음 이슈(파킹 중)를 파일럿에서 자연 포착 → 진단 문구(N초·KB) 확보 기회

## 5. 워크스트림 B — 주간 운영 (1~4주)

### 0주 킥오프 (딥 세션, ~1시간)
1. 김도은 하루 업무 관찰 → "기록이 가장 힘든/누락되는 순간" Top 1~2 특정
2. 그게 Carelog 범위 안인지 합의 (범위 밖이면 솔직히 제외)
3. 멀티 기기 세팅(워크스트림 A) 동반 완료
4. 성공 기준·톤 합의: **"'못 쓰겠다'가 정답일 수도 있다, 그걸 찾는 게 네 일"**

### 1~4주 주간 1시간 (고정 구조)
- **15분** — 지난주 개선안이 실제로 나아졌는지 확인
- **30분** — 실제 상담 기록 1건을 **옆에서 관찰**(탭 수·머뭇거림 기록). *의견보다 관찰이 진실*
- **15분** — 다음 주 우선순위 1개 합의

### 매주 산출 규칙
> **"이번 주 마찰 Top 3 + 다음 회의 전 최소 1개 개선 배포"**

이 약속이 주간 회의 비용을 정당화하고, spec-driven 흐름(피드백 → 백로그 카드 → spec → 배포)에 그대로 얹힌다.

## 6. 과부하 가드레일 (대표 우려 반영)

> **제품이 김도은의 일을 *줄여야지*, '테스트'라는 일을 *추가*하면 실패다.**

- **추가 과업 0** — 설문지·사용일지 강제 금지(과부하의 함정). 평소처럼 쓰고 주 1시간만 대화
- **분석 부담은 다온/대표가** — 탭 수·실패율·건수는 계측·관찰로 우리가 짊어진다
- **경량 비동기는 선택** — 받는다면 하루 끝 1줄("오늘 편했음/짜증") 정도, 강제 아님
- **4주 캡 + 중도 축소** — 끝이 정해져 부담이 안 됨. 힘들어하는 신호 시 즉시 줄임
- 위계 안전장치 — 가능하면 원장 빠진 자리에서 솔직한 불편을 끌어냄

## 7. 피드백 → 개선 파이프라인

회의에서 나온 개선안은 즉석 메모로 끝내지 않고 **백로그 카드**로 떨군다.
→ 양식: `specs/000-backlog/pilot-feedback-card-template.md`
→ 우선순위·범위 판단 후 작은 건 바로 배포, 큰 건 spec으로 승격.

## 8. 개인정보 · 동의 (헌법 I)

- 실제 환자 상담을 다루므로 **녹음·기록 동의**는 기존 진료 시점 캡처 흐름을 따른다(환자정보 동의와 분리)
- 피드백 메모·관찰 기록에 **환자 식별정보를 적지 않는다**(체어명·상황만)
- 공용 기기 화면 자동잠금(§4)으로 노출 최소화
- 실시간 알림은 이미 `chair_audit_logs`(PII 0) 구독이라 전송선에 환자정보 없음
