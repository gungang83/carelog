# Carelog 요금·등급 정책 (초안)

> **상태**: 초안 — 핵심 방향 확정(2026-06-19, 대표님). 가격(원)·결제연동은 미정.
> **소유**: 다온(PM). 이 문서가 SSOT. 음성 보존·재청취 구현은 `specs/009-audio-archive`가 이 게이트를 참조.

## 0. 핵심 원칙
- **무료도 핵심 와우(녹음 → AI 상담기록)는 충분히 경험**한다. 진입장벽을 낮춰 확산.
- **비용 유발 요소만 제한/과금**한다 (저장·SMS·AI 실비).
- **텍스트 차트(의무기록)는 등급과 무관하게 보존**한다. 등급 레버는 *음성 원본·부가기능*에만 건다.

## 1. 과금 단위
- **병원(기관) 단위 월정액.** (직원 seat·사용량 단위 아님 — 치과 영업에 단순·예측가능)
- **사용량 추가 과금(토큰)은 이후 단계 도입.** 지금은 골격만 잡고 실제 빌링은 나중.

## 2. 등급 (Free / Pro / Clinic)

| 기능 | **Free** | **Pro** | **Clinic** |
|---|---|---|---|
| 녹음 → AI 변환·텍스트 기록 | ✅ (핵심, 넉넉히) | ✅ 무제한 | ✅ 무제한 |
| **텍스트 상담기록 보존** | ✅ 영구 | ✅ 영구 | ✅ 영구 |
| **음성 원본 보존** | **최근 3개만**(롤링) | **90일** | **1년+ / 설정 가능** |
| **음성 재청취** | 최근 3개만 | ✅ 직원 | ✅ + **감사로그(누가 들었나)** |
| 이미지·그림 주석 / 다기기 실시간 알림 | ✅ | ✅ | ✅ |
| 전체 복사(덴트웹 붙여넣기) | ✅ | ✅ | ✅ |
| 멤버 수 | 소수 | 무제한 | 무제한 + 다지점 |
| 상담 분석·CRM(성장축) | ❌ | 일부 | ✅ |
| 환자 SMS 전달 / AI 사용량 | 토큰(아래) | 토큰(아래) | 토큰(아래) |

### Free의 "최근 3개" 음성 룰 (후킹 포인트)
- 기관별 **가장 최근 음성 원본 3개만** 보존·재청취. 4번째 저장 시 **가장 오래된 원본 자동 삭제**.
- **텍스트 기록은 그대로 유지**(음성만 롤링). → "원본을 더 남기고 싶다 = 유료" 전환 동기.

## 3. 토큰 (사용량 과금) — 실비 항목 분리
실제 비용이 발생하는 기능은 등급과 별개로 **토큰(사용량)**으로 과금한다.
- **대상**: ① 환자 SMS 발송(Solapi 실비) ② AI 전사·요약(OpenAI/Anthropic 실비) ③ (선택) 저장 용량 초과분.
- **초기 정책**: 가입 시 **무료 토큰 지급** → 소진 후 충전.
- **도입 시점**: 이후 단계(지금은 정책만 명시, 빌링 미구현).

## 4. 내부 / 파트너
- 예미안치과·케어로그 치과·파일럿(도은쌤) = **Clinic 등급 무상**.
- 가격은 외부 확산 시 적용.

## 5. 미결 / 다음
- 가격(원) 책정.
- 무료 핵심기능 한도(녹음·기록 건수) — 권장: 무제한, 비용유발만 토큰.
- 결제·빌링 연동(토큰 충전 포함) = 향후 별도 스펙.
- ⚖️ **음성 원본 보존의 법적 검토** — 의무기록은 텍스트 차트이고 음성은 부가 백업이라 등급화 가능하나, 의료법상 음성 보존 의무 여부 별도 확인.

## 6. 구현 연결
- 기관(`institutions`)에 **`plan`(free | pro | clinic)** 필드 추가 → 기능 게이트의 단일 출처.
- 음성 보존기간·재청취 권한·"최근 3개" 룰 = `specs/009-audio-archive`에서 이 `plan`을 참조해 구현.
- 토큰 잔액·과금은 별도(향후) — 본 문서는 정책, 빌링 시스템은 추후.
